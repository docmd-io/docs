Die `docmd` Version 0.7.6 führt eine **einheitliche URL-Normalisierungs-Engine** ein, die 301-Weiterleitungen eliminiert, URLs mit abschließenden Schrägstrichen (Trailing Slash) für SEO erzwingt und eine konsistente Link-Handhabung auf allen Ebenen bietet — Markdown, Buttons, Tags, Navigation und Menüleiste. Sie enthält zudem gezielte Fehlerbehebungen für SPA und die Entwicklererfahrung.

## 🔗 Einheitliche URL-Normalisierung

Die gesamte URL-Verarbeitung erfolgt nun über eine einzige `resolveHref()`-Funktion in `@docmd/parser`. Die Erzwingung von abschließenden Schrägstrichen, das Entfernen von `index.md` und die Vermeidung von Doppelschrägstrichen werden konsistent angewendet auf:

- Markdown-Links
- Button- & Tag-Container
- Navigations- & Menüleisten-Konfiguration
- Auto-Router (Zero-Config-Modus)
- Hreflang- und kanonische `<link>`-Tags

**Wichtige Verbesserungen:**
- Keine 301-Weiterleitungsketten mehr — `/configuration/overview` → `200` direkt bei `/configuration/overview/`
- Keine `/index`-Artefakte oder `.md`-Erweiterungen in gerenderten Links
- Hreflang-Alternate-Tags entfernen nun korrekt die Locale-Präfixe von Pfaden

Siehe den Leitfaden [Verlinkung & Referenzierung](https://docs.docmd.io/de/content/syntax/linking/) für die vollständige URL-Übersetzungstabelle.

## 🆕 Universelle Präfixe `external:` und `raw:`

### `external:` — In neuem Tab öffnen erzwingen

Fügen Sie jedem Link das Präfix `external:` hinzu, um ihn in einem neuen Tab zu öffnen. Funktioniert in Markdown, Buttons, Tags, Navigation und Menüleiste:

```markdown
[In neuem Tab öffnen](external:./configuration/overview.md)
::: button "API-Dokumentation" external:./api/node-api.md
```

**Verhaltensänderung:** HTTP/HTTPS-Links werden nicht mehr automatisch in einem neuen Tab geöffnet. Verwenden Sie explizit `external:`, wenn Sie dieses Verhalten wünschen — dies gibt Ihnen die volle Kontrolle, auch über Links zu Ihren eigenen externen Dokumentationsseiten.

### `raw:` — Normalisierung umgehen

Verlinken Sie auf eine rohe `.md`-Datei, ohne dass die Engine deren Erweiterung entfernt:

```markdown
[Vorlage herunterladen](raw:templates/starter.md)
```

Beide Präfixe werden aus dem gerenderten HTML entfernt — sie sind reine Signale für die Build-Zeit.

Siehe den Leitfaden [Verlinkung & Referenzierung](https://docs.docmd.io/de/content/syntax/linking/) für Details.

## 🧩 Zentralisierte URL-API für Plugins

Eine neue URL-Utility-Ebene in `@docmd/parser` wird über `@docmd/api` erneut exportiert. Plugins müssen den `outputPath` nicht mehr manuell parsen — die Engine berechnet saubere URLs für jede Seite vor:

```typescript
import { outputPathToSlug, sanitizeUrl } from '@docmd/api';

export async function onPostBuild({ pages }) {
  for (const page of pages) {
    page.urls.slug;      // "guide/"
    page.urls.canonical; // "https://example.com/guide/"
    page.urls.pathname;  // "/guide/"
  }
}
```

Siehe den Leitfaden [Plugin-Entwicklung](https://docs.docmd.io/de/advanced/plugins/) für die vollständige API-Referenz.

## 🔧 Entwicklererfahrung

### Lautlose Plugin-Warnungen im Entwicklungsmodus

Zuvor gab ein fehlendes oder falsch konfiguriertes Plugin (z. B. `@docmd/plugin-math`) bei **jeder Dateiänderung** während `docmd dev` eine Warnung aus:

```
↻ Änderung in docs/en/guide.md... ⚠️  Konnte Plugin nicht laden: @docmd/plugin-math
Fertig.
```

Plugin-Warnungen werden nun **einmal pro Entwicklungssitzung** ausgegeben und bei nachfolgenden Rebuilds unterdrückt. Die erste Warnung erscheint weiterhin, damit Sie wissen, dass etwas nicht stimmt — sie überflutet nur nicht mehr das Terminal.

### SPA-Hash-Navigation korrigiert

Im SPA-Modus schlägt das Klicken auf einen `#heading`-Anker auf der aktuellen Seite nicht mehr stillschweigend fehl. Zuvor wurde nach einer `pushState`-Navigation das native Scrollen zum Anker des Browsers nicht ausgelöst. Der SPA-Router fängt nun Hash-Klicks auf derselben Seite ab und scrollt reibungslos zum Zielelement:

```markdown
[Zu Konfiguration springen](#configuration)  <!-- funktioniert jetzt im SPA-Modus -->
```

Das Vor- und Zurückblättern im Browser durch die Hash-Historie (via `popstate`) scrollt ebenfalls korrekt.

## 🚀 SEO-Verbesserungen

- Alle internen URLs werden mit abschließenden Schrägstrichen gerendert — eliminiert 301-Weiterleitungsketten
- Kanonische und Hreflang-`<link>`-Tags verwenden korrekte, saubere Pfade
- Keine `.md`-Erweiterungen oder `/index`-Artefakte im Produktions-HTML

## Migrationsleitfaden

Aktualisieren Sie mit `npm update @docmd/core` oder `pnpm update @docmd/core`. Keine Konfigurationsänderungen erforderlich.

::: callout info
Wenn Sie benutzerdefinierte serverseitige Weiterleitungsregeln haben, die abschließende Schrägstriche entfernen, entfernen Sie diese. Die Engine erzeugt nun nativ das kanonische Format mit abschließendem Schrägstrich.
:::
