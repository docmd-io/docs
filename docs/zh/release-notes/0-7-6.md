`docmd` 0.7.6 版本引入了 **统一 URL 规范化引擎**，消除了 301 重定向，强制执行用于 SEO 的尾随斜杠 URL，并为所有界面带来了始终如一的链接处理。

## ✨ 特性亮点

### 🔗 统一 URL 规范化

现在所有 URL 处理都通过 `@docmd/parser` 中的单个 `resolveHref()` 函数进行。这确保了尾随斜杠强制执行、`index.md` 剥离和双斜杠防止在 Markdown 链接、按钮容器和导航菜单中得到完全一致的应用。

- **消除 301 重定向**: 像 `/guide/overview` 这样的 URL 现在在构建时直接解析为 `/guide/overview/`，节省了一次网络往返。
- **纯净路径解析**: 在任何渲染的链接中不再有 `/index` 残留或 `.md` 扩展名。
- **全局一致性**: 同样的逻辑适用于自动路由、hreflang 标签和规范链接。

### 🆕 通用的 `external:` 和 `raw:` 前缀

我们引入了两个新的魔法前缀，让你能够完全控制所有 `docmd` 组件中的链接行为。

- **`external:` — 强制新标签页打开**: 在任何链接（Markdown、按钮、菜单栏）中添加此前缀，以强制其在新标签页中打开。
- **`raw:` — 绕过规范化**: 当你需要链接到一个文件（如 `template.md`）而又不希望引擎剥离其扩展名时，请使用此前缀。

### 🧩 插件的集中式 URL API

插件不再需要手动解析 `outputPath`。我们通过 `@docmd/api` 公开了一个新的 URL 工具层，在构建过程中为每个页面提供预先计算的、纯净的 URL。

## 🐛 错误修复与 UX

### ⚡ SPA 哈希导航修复
在 SPA 模式下，单击当前页面上的 `#heading` 锚点不再静默失败。SPA 路由器现在会拦截同页面的哈希点击，并平滑地滚动到目标元素。

### 🔕 静默插件警告
缺失或配置错误的插件现在在 **每个开发会话中仅打印一次** 警告。在 `docmd dev` 的后续重建中将抑制这些警告，以保持终端整洁。

### 🚀 SEO 改进
- **尾随斜杠**: 默认情况下，所有内部 URL 现在都以尾随斜杠渲染，以遵循 SEO 最佳实践。
- **规范路径清晰化**: 规范链接和 hreflang `<link>` 标签现在会自动使用正确的、规范化后的路径。

## 迁移指南

使用 `npm update @docmd/core` 或 `pnpm update @docmd/core` 更新到最新版本。

::: callout tip "服务器配置"
如果你有剥离尾随斜杠的自定义服务器端重定向规则，你应该将其删除，因为引擎现在原生生成规范的尾随斜杠格式。
:::
