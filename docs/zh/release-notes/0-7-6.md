`docmd` 0.7.6 版本引入了 **统一 URL 规范化引擎**，消除了 301 重定向，强制执行用于 SEO 的尾随斜杠 URL，并在每个层面（Markdown、按钮、标签、导航和菜单栏）带来了一致的链接处理。它还发布了针对性的 SPA 和开发者体验修复。

## 🔗 统一 URL 规范化

现在所有 URL 处理都通过 `@docmd/parser` 中的单个 `resolveHref()` 函数进行。尾随斜杠强制执行、`index.md` 剥离和双斜杠防止一致地应用于：

- Markdown 链接
- 按钮和标签容器
- 导航和菜单栏配置
- 自动路由（零配置模式）
- Hreflang 和规范 `<link>` 标签

**主要改进：**
- 不再有 301 重定向链 —— `/configuration/overview` 直接以 200 状态跳转到 `/configuration/overview/`
- 在任何渲染链接中不再有 `/index` 残留或 `.md` 扩展名
- Hreflang 备用标签现在可以正确地从页面路径中剥离语言环境前缀

请参阅 [链接与引用](https://docs.docmd.io/zh/content/syntax/linking/) 指南以获取完整的 URL 转换表。

## 🆕 通用的 `external:` 和 `raw:` 前缀

### `external:` — 强制新标签页打开

在任何链接中添加 `external:` 前缀以在新标签页中打开它。适用于 Markdown、按钮、标签、导航和菜单栏：

```markdown
[在新标签页中打开](external:./configuration/overview.md)
::: button "API 文档" external:./api/node-api.md
```

**行为变更：** HTTP/HTTPS 链接不再自动在新标签页中打开。当你需要该行为时，请显式使用 `external:` —— 这赋予你完全的控制权，包括指向你自己外部文档站点的链接。

### `raw:` — 绕过规范化

链接到原始 `.md` 文件，而引擎不会剥离其扩展名：

```markdown
[下载模板](raw:templates/starter.md)
```

这两个前缀都会从渲染后的 HTML 中剥离 —— 它们仅作为构建时的信号。

请参阅 [链接与引用](https://docs.docmd.io/zh/content/syntax/linking/) 指南了解详情。

## 🧩 插件的集中式 URL API

`@docmd/parser` 中新的 URL 实用程序层通过 `@docmd/api` 重新导出。插件不再需要手动解析 `outputPath` —— 引擎会为每个页面预先计算干净的 URL：

```typescript
import { outputPathToSlug, sanitizeUrl } from '@docmd/api';

export async function onPostBuild({ pages }) {
  for (const page of pages) {
    page.urls.slug;      // "guide/"
    page.urls.canonical; // "https://example.com/guide/"
    page.urls.pathname;  // "/guide/"
  }
}
```

请参阅 [插件开发](https://docs.docmd.io/zh/advanced/plugins/) 指南以获取完整的 API 参考。

## 🔧 开发者体验

### 开发模式下的静默插件警告

以前，缺失或配置错误的插件（例如 `@docmd/plugin-math`）会在 `docmd dev` 期间的 **每次文件更改** 时打印警告：

```
↻ Change in docs/en/guide.md... ⚠️  Could not load plugin: @docmd/plugin-math
Done.
```

插件警告现在在每次开发会话中 **仅打印一次**，并在随后的重建中被抑制。第一个警告仍然会出现，以便你知道出了问题 —— 它只是不会充斥终端。

### SPA 哈希导航修复

在 SPA 模式下，单击当前页面上的 `#heading` 锚点不再静默失败。以前，在 `pushState` 导航后，浏览器的原生“滚动到锚点”不会触发。SPA 路由器现在拦截同页面的哈希点击，并平滑地滚动到目标元素：

```markdown
[跳转到配置](#configuration)  <!-- 现在在 SPA 模式下可以工作 -->
```

通过哈希历史记录进行的浏览器后退/前进（通过 `popstate`）也能正确滚动。

## 🚀 SEO 改进

- 所有内部 URL 渲染时带有尾随斜杠 —— 消除 301 重定向链
- 规范和 hreflang `<link>` 标签使用正确的干净路径
- 生产环境 HTML 中零 `.md` 扩展名或 `/index` 残留

## 迁移指南

使用 `npm update @docmd/core` 或 `pnpm update @docmd/core` 进行更新。无需更改配置。

::: callout info
如果你有剥离尾随斜杠的自定义服务器端重定向规则，请将其删除。引擎现在原生生成规范的尾随斜杠格式。
:::
