`docmd` 0.7.6 版本引入了**统一 URL 规范化引擎**，消除 301 重定向，强制使用尾部斜杠 URL 以优化 SEO，并在所有表面——Markdown 内容、按钮容器、标签和导航配置中实现一致的链接处理。两个新的通用链接前缀 `external:` 和 `raw:` 让作者可以精确控制链接行为，无需学习特定容器的语法。

## 🔗 统一 URL 规范化引擎

### 问题

在 0.7.6 之前，URL 处理分散在六个不同的代码位置。每个位置都有自己的内联逻辑，存在微妙的不一致性：

- **缺少尾部斜杠** — 内部链接渲染为 `/configuration/overview` 而不是 `/configuration/overview/`，导致 Web 服务器发出 301 重定向。
- **`index.md` 残留** — 如 `localisation/index.md` 这样的链接渲染为 `localisation/index` 而不是干净的 `localisation/`。
- **容器盲点** — 按钮容器和标签将 URL 原样传递，不做任何规范化。
- **双斜杠风险** — 如果用户不小心多写了一个斜杠，没有防护措施阻止 `foo//bar/`。

### 修复方案

所有 URL 处理现在都通过 `@docmd/parser` 中的单一 `resolveHref()` 函数进行路由。

### URL 转换表

| 输入 | 输出 | 应用的规则 |
|:---|:---|:---|
| `overview.md` | `overview/` | 去除 `.md`，添加尾部 `/` |
| `overview` | `overview/` | 添加尾部 `/` |
| `overview/` | `overview/` | 已是规范格式 |
| `localisation/index.md` | `localisation/` | 去除 `index.md` → 文件夹路径 |
| `../index.md` | `../` | 父目录索引 |
| `overview.md#settings` | `overview/#settings` | 哈希片段保留 |
| `foo//bar.md` | `foo/bar/` | 双斜杠合并 |
| `https://example.com` | `https://example.com` | 外部链接 — 不修改 |
| `external:overview.md` | `overview/` *(+ 新标签页)* | 前缀剥除，标志设置 |
| `raw:docs/readme.md` | `docs/readme.md` | 绕过规范化 |

## 🆕 通用 `external:` 前缀

新的 `external:` 前缀现在在**所有**可以编写链接的地方都被识别——Markdown、按钮、标签以及任何渲染链接的容器。

```markdown
<!-- 强制在新标签页中打开 -->
[打开设置](external:./configuration/overview.md)

<!-- HTTP 链接自动检测为外部链接 -->
[GitHub](https://github.com/docmd-io/docmd)

<!-- 在按钮中也有效 -->
::: button "API 文档" external:./api/node-api.md icon:external-link
```

## 🆕 `raw:` 前缀 — 绕过规范化

由于引擎现在会规范化所有 `.md` 扩展名，需要链接到原始 `.md` 文件的作者可以使用 `raw:` 前缀：

```markdown
[下载模板](raw:templates/starter.md)

::: button "获取源码" raw:docs/readme.md icon:download
```

## 🚀 SEO 改进

所有内部页面现在一致地使用尾部斜杠 URL 渲染。这消除了之前 Web 服务器自动附加斜杠时产生的 301 重定向链。

## 📖 文档

[链接与引用](../content/syntax/linking.md) 指南已全面重写。文档在所有三个语言区域（英语、中文、德语）中可用。

## 🔍 全面链接错误修复

我们解决了在整个生态系统进行严格测试期间发现的几个边缘情况链接行为：

- **搜索客户端路径**：搜索 UI 现在在构建最终 `linkHref` 时显式去除前导斜杠，防止在导航到根索引或标题片段时出现双斜杠（例如 `//`）。
- **菜单栏规范化**：菜单栏配置（`config.menubar.left` 和 `config.menubar.right`）现在使用核心 URL 规范化引擎（`resolveHref`），确保菜单栏链接遵循与 markdown 链接完全相同的尾部斜杠规则和 `external:` 前缀处理。这消除了顶层导航上的 301 重定向。
- **目录 (TOC) 和版本切换器双斜杠免疫**：通过防止搜索客户端用 `//` 污染地址栏，依赖于相对哈希链接的目录生成器和保留当前路径名的版本切换器不再继承或传播双斜杠。版本切换器还获得了针对双斜杠的独立正则表达式保护。
- **站点徽标链接**：站点标题和徽标链接已更新为指向目录根（`./`），而不是显式链接到 `index.html`，消除了干净 URL 结构中的例外情况。

## 迁移指南

对于**最终用户**：使用 `npm update @docmd/core` 或 `pnpm update @docmd/core` 更新到 0.7.6。无需配置更改——规范化完全向后兼容。
