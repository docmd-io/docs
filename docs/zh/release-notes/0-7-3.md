`docmd` 0.7.3 版本专注于视觉一致性和生态系统扩展。我们统一了所有交互式容器的图标渲染逻辑，并增加了对 Bun JavaScript 运行时的全面支持。

## ✨ 通用容器图标 (Isomorphic Container Icons)

我们重构了核心解析器，采用了共享的**容器完整性引擎 (Container Integrity Engine)**。这确保了所有交互式元素——提示框 (Callouts)、卡片 (Cards)、折叠面板 (Collapsibles)，以及现在的**标签页 (Tabs)**——都支持统一的 `icon:name` 语法。

### 标签页中的图标

现在，您可以直接在标签页标签中添加任何 Lucide 图标，为用户提供即时的视觉上下文。这在区分包管理器、操作系统或编程语言时非常有用。

```markdown
::: tabs
== tab "npm" icon:box
npx @docmd/core dev

== tab "Bun" icon:zap
bunx docmd dev
:::
```

所有图标均渲染为经过优化的内联 SVG，保持了 `docmd` 对零布局偏移和百毫秒级导航性能的承诺。

## ⚡ 一流的 Bun 支持

随着 Bun 运行时的快速普及，我们更新了所有核心文档和“入门指南”，加入了原生的 Bun 示例。

- **`bunx` 支持**：所有一次性命令现在都包含与 `npx` 并列的 `bunx` 替代方案。
- **依赖管理**：安装指南现在涵盖了用于项目级集成的 `bun add` 工作流。
- **文档同步**：英文、中文和德文指南已完成同步，确保无论您使用哪种运行时或语言，都能获得一致的体验。

## 🛠️ 内部重构

此版本包含了对 `docmd/parser` 包的一次重大架构清理：

- **重构实用程序层**：将 `parseTitleAndIcon` 移至中央 `container-helper` 实用程序中。这消除了 `tabs` 和 `common-containers` 模块之间的逻辑重复。
- **性能优化**：共享助手采用优化的正则模式，减少了解析包含大量交互式容器的大型文档时的开销。

## 📝 完整变更日志

### 🚀 功能与增强
- **标签页图标**：在 `== tab` 子分隔符中增加了对 `icon:name` 语法的支持。
- **共享解析逻辑**：创建了统一的 `container-helper` 实用程序，用于解析所有容器的标题和图标。
- **Bun 生态系统**：全面更新了所有入门指南和安装指南，加入 Bun/bunx 示例。
- **文档审核**：完成了整个 `guides/` 系列的 v0.7.2 审核及多语言（中/德）本地化。
- **视觉一致性**：标准化了标签导航栏中的图标大小和对齐方式。

### 🐛 缺陷修复
- **导航同步**：修复了中文 (`zh`) 导航中一个遗留的“实用技巧”部分错误显示的问题。
- **解析器健壮性**：修复了一个边缘情况，即标签页内的嵌套容器在包含特定的 Markdown-it 围栏标记时偶尔无法渲染的问题。
- **语言包一致性**：纠正了德文 `content-ux` 指南中的几处技术翻译错误。

## 迁移指南

对于**终端用户**：无破坏性变更。您现在可以开始在现有标签页中添加图标，无需任何配置更改。

对于**插件作者**：如果您之前手动解析标签页页眉，我们建议切换到 `@docmd/parser` 导出的 `parseTitleAndIcon` 实用程序，以保持与未来图标增强功能的兼容性。
