`docmd` 0.7.2 版本正式引入了备受期待的 **`docmd deploy`** 命令。这个全新的实用工具旨在架起“构建站点”与“专业托管”之间的桥梁，助您轻松进军生产环境。

## ✨ 亮点

### 🛡️ 引入全新的 'deploy' 命令

`docmd` 首次提供内置的生产级托管脚手架。您不再需要手动编写复杂的服务器配置，现在可以即刻生成经过安全强化的优化模板：
- **Docker**：优化的多阶段构建，具备精确的层缓存 (Layer Caching) 和版本锁定，确保 100% 可重复构建。
- **NGINX & Caddy**：提供生产就绪的配置，默认包含行业标准的安全性响应头（如 `X-Content-Type-Options: nosniff`, `X-Frame-Options: SAMEORIGIN`）。
- **开箱即用的性能**：自动配置 GZIP 压缩，并为静态资源制定了激进的“不可变 (Immutable)”缓存策略。
- **SPA 弹性**：为所有目标预配置了智能的 404 处理机制和 SPA 路由回退。

### ⚠️ 安全优先与幂等性

尽管这是一个新功能，但我们在设计时就考虑到了成熟度。`deploy` 命令具有 **幂等性**，这意味着它绝不会意外覆盖您的文件。只有在您明确希望使用我们的最新模板重新生成配置时，才需要使用 `--force` 标志。

### 🛡️ 7 大支柱 Failsafe 验证

为了确保此版本的稳如磐石，我们的内部验证引擎已重构为 **7 个逻辑稳定性支柱**。其中包括全新的 **动态完整性引擎 (Dynamic Integrity Engine)**，它能即时捕捉整个 Monorepo 中的版本不匹配问题。

### 🌍 全球生态系统扩展

我们正通过重大的本地化更新来扩展全球影响力：
- **德语文档上线**：完整的文档套件现已支持德语 (`/de/`)。
- **优化的中文文档**：对现有的中文文档 (`/zh/`) 进行了深度的语义优化和校对。
- **多语言 README**：仓库核心 README 现已支持**德语、西班牙语、日语、法语和中文**，让 docmd 能够触达全球更多的开发者。

## 📝 完整更新日志

### 🚀 功能与增强
- **新命令**：引入 `docmd deploy`，支持 `--docker`, `--nginx`, `--caddy` 和 `--force` 标志。
- **Docker 脚手架**：通过 `package.json` 层缓存和核心版本锁定自动化生成 Dockerfile。
- **Web 服务器脚手架**：通过安全头、GZIP 和智能 404/SPA 路由强化了 NGINX 和 Caddy 模板。
- **Failsafe**：将验证逻辑合并为更快、更专业的 7 阶段流水线。
- **版本控制**：全新的完整性引擎会自动标记过时的内部工作区引用。
- **SEO 插件**：在社交媒体 (OG/Twitter) 元数据中完全遵循 `titleAppend` frontmatter 属性。
- **本地化**：新增了完整的**德语**文档翻译，并对**中文**文档进行了全面优化。
- **项目 README**：为仓库 README 提供了多语言支持（德语、西班牙语、日语、法语和中文）。

### 🐛 问题修复
- **Caddy 配置**：修正了 Caddyfile 兼容性的 Cache-Control 头部语法（添加引号）。
- **Bump 脚本**：修复了 `scripts/bump.js` 中的正则 `lastIndex` 漏洞，该漏洞曾导致版本替换不一致。
- **CLI 安全**：为所有部署命令入口点添加了适当的异步错误处理。
- **在线编辑器**：修复了移动端溢出问题，并改进了模板切换的状态追踪。

## 迁移指南

**最终用户**：无破坏性更改。只需更新并尝试全新的 `docmd deploy` 命令，感受将站点打包发布到生产环境是多么简单。

**插件作者**：无需任何操作。
