## 问题
个人本地控制经常遇到不同版本引起的缓存或者库错误并随着管理员无法访问其计算机而停顿等异常中断或者单点故障事故。

## 为什么重要
CI Continuous Delivery 的敏捷化理念使得变更直接几分内实现线上交付而不用人为的过多进行冗杂等待与人工操作交互避免操作差池与错误指令出现风险的频发展露无余暴露缺陷。"

## 方法
采用由 Github 提供的一个自动化实例 Linux Ubuntu 来克隆代码执行并安全通过其自身的服务协议通过无秘钥上传部署的方式解决发布路径的权限等问题及保障执行过程与一致稳定的云环境。"

## 实施
你可以写入一个 yaml 文件包含 checkout 再到 setup 步骤执行构建完毕并依赖第三方著名的稳定集成组件比如 peaceiris 搬移静态化成果物输出将其放入配置项下分支进行在线服务运行并对外提供访问服务端口开启并解决对应权限与参数问题确保路径符合要求不发生偏差等细节问题的校验确认保障无虞等。"

## 权衡
你的自由度和权限被绑定受限于宿主主机的条款限制服务稳定性如果服务商宕机崩溃将直接连累阻截你的发布等一系列相关的外卡项不定的因数等因素。不过这也是非常少见基本不影响大规模运用的情况。"
