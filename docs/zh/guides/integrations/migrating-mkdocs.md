## 问题
大量的前端项目苦于无法或者极为头痛地需要专门建立繁琐多变的 Python pip 环境来支持构建系统运行和维护造成断层割裂。

## 为什么重要
这使得开发者环境的融合达成共识能极大的改善整个公司层面统一全链路工作流工具栈的管理并且对于持续集成非常有效与便捷的维护方式提供方便通道与管理路径便利以及快捷方法指导和建议意见给出并参考学习和借鉴实施路径。"

## 方法
我们大量类似和支持着他们曾经优秀的体验等比如其优秀的标记提醒框设计等以及多标签页面设计均支持良好的过度迁移与兼容使用保障不产生割裂。并将原来的配置文件重写为极简 JS 等格式。

## 实施
原有的各类插件由于生态的原因其实在这边均有对应功能的吸收并固化成为了系统能力所以不必要引入第三方依赖就可以直接书写和享用并且原有的一些书写习惯比如特殊的标记语法直接也使用查找工具一两秒内完成全局更改就能正常使用呈现了。"

## 权衡
不可否认人家积累多年的各种特别极其冷门特殊的插件比如什么生成特殊的某个特殊图谱等等功能本系统因为架构以及专注度差异暂不如它丰富全面但这也在持续改进并可以通过自身插件机制由开发者参与丰富这个平台从而慢慢填补空白弥补缺憾。"
