---
title: "管理 docmd 的多版本文档 (v1, v2)"
description: "关于多版本文档的综合指南。"
---

## 问题
不断淘汰产品的 v1 版本文档对使用 LTS 滞后版本的企业极不友好，易造成操作事故。

## 为什么重要
老旧客户读到新指令应用到老环境将带来灾难性的报错工单。

## 方法
启用内置多级版本控制器，使版本间地址保留并切换。

## 实施
将即将归档的源放到独立的 `docs-v1.x/` 下。使用 `docmd.config.js` 内的 `versions` 选项声明这些存档的物理路由。

## 权衡
仓库的 git 历史以及物理大小会被旧版本的代码大幅度膨胀。
