## 问题
随机长度并杂糅主旨的杂乱段落在分片（Token Chunking）中容易断章取义截断上下文。

## 为什么重要
丢失伴生断裂的补充语句使得逻辑直接反向失效误导使用者系统发生灾难性毁灭（例如强制抹除脚本少提醒）。

## 方法
物理聚集操作与逻辑边界依靠特定组件相连。

## 实施
Header 划分必须严格承载一条完整执行体。保证代码块与对应的提示语句和 `callouts` 的物理距离几乎为 0。

## 权衡
被限制的框架结构扼杀了讲述悠长故事文档的浪漫文学性。
