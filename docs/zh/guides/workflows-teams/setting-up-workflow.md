---
title: "在 docmd 中设立规范的团队发布协同操作流程"
description: "一个完整的协同开发指南规训设定大纲。"
---

## 问题
毫无约束的分离文档管理很容易引发修改忘记录入或者内容格式完全杂乱成堆以至于技术编辑更多的时间在擦屁股和排版合并而没在书写内容上。

## 为什么重要
一旦没有流程规范就会产生严重的发放迟滞最后长期得过且过而直接腐烂并陷入僵局与困境乃至无法运转被废弃等结局。"

## 方法
将代码审查纳入到部署并与应用解耦。依赖零配置来构建 Docs as code 也就是作为代码存在的文档审核流水线规范。

## 实施
依靠 Git 系统的 Hooks 例如 Husky 等设置约束 `docmd build --strict` 防止无效链接提交进而阻碍上线部署流程并约束团队的操作动作机制等全方面保障。"

## 权衡
非技术的协作者如业务和产品线人员会被这种类似技术底层的屏障所劝退。必须考虑让他们在 GitHub 网页页面内利用 GUI 修改并作为妥协。"
