The `docmd` 0.7.0 release is a landmark milestone — introducing native multi-language support, a runtime theme engine, a safer plugin architecture with validation descriptors, and a fully modular package structure that lets you embed the build pipeline in your own tools.

## ✨ Highlights

### 🌍 Internationalisation (i18n)
docmd now has first-class i18n support. Configure multiple locales and docmd builds a complete, localised version of your site for every locale — with locale-first URLs (`/hi/`, `/zh/`), translated UI strings, and native RTL layout support.

```js
i18n: {
  default: 'en',
  locales: [
    { id: 'en', label: 'English', dir: 'ltr' },
    { id: 'hi', label: 'हिन्दी', dir: 'ltr' },
    { id: 'zh', label: '中文', dir: 'ltr' },
    { id: 'ar', label: 'العربية', dir: 'rtl' },
  ]
}
```

### 🎨 Dynamic Theme Engine
A new CSS custom-property–based theming system lets you define and switch themes at runtime without a full page reload. Themes can be declared in config or registered dynamically by a plugin, giving you complete control over the visual experience without rebuilding.

### 🔌 Plugin Validation Descriptors
Plugins can now declare a JSON Schema descriptor so docmd validates plugin options at config-load time and surfaces clear, actionable errors before the build starts — no more cryptic runtime failures from a misconfigured plugin.

### 📦 Core Plugin Extraction
Built-in features (search, SEO, sitemap, PWA, LLMs, math, mermaid, threads) have been refactored into explicit `@docmd/plugin-*` packages. You opt in to exactly what you need rather than having everything bundled — resulting in faster builds and a smaller output.

### 🧩 Modular Package Architecture
The internal build pipeline has been split into focused packages: `@docmd/core`, `@docmd/parser`, and `@docmd/live`. This lets you embed the docmd engine directly in your own tools, CI pipelines, or custom build scripts.

## 📝 Complete Changelog

### 🌍 i18n Architecture
- **Locale-first URL generation**: Build loop now nests `buildLocales` → `buildVersions` → pages for each configured locale.
- **Translated UI strings**: EJS templates resolved against JSON locale files (`en.json`, `hi.json`, `zh.json`, etc.) — all template strings accessible via `t('key')` helper.
- **RTL layout support**: `dir` attribute on `<html>` driven by locale config — RTL locales get mirrored sidebar and nav layout automatically.
- **Zero-footprint fallback**: Sites with no `i18n` block build identically to pre-0.7.0 — no path or output changes.

### 🎨 Theme Engine
- **CSS custom property tokens**: Theme definitions compile to a single `<style>` block of CSS variables injected at build time.
- **Runtime switching**: Theme switcher UI registers a `docmd:theme-change` client event — no page reload required.
- **Plugin-registered themes**: Plugins can call `ctx.registerTheme(descriptor)` to add themes dynamically at build time.

### 🔌 Plugin System
- **Validation descriptors**: Added `descriptor` export to plugin spec — JSON Schema validated at config-load via `ajv` before any build step runs.
- **Shorthand reservation extended**: All `@docmd/plugin-*` shorthand names are now explicitly reserved; third-party plugins require full package names.
- **`ctx.registerTheme` hook**: New plugin lifecycle hook for registering theme descriptors.

### 🏗️ Core & Parser
- **`@docmd/parser` extraction**: Markdown pipeline (remark + rehype) extracted into a standalone package consumable independently of the full build engine.
- **`@docmd/live` extraction**: Dev server and live-reload logic moved to `@docmd/live`, reducing the `@docmd/core` install footprint.
- **Per-version `navigation.json`**: Navigation can now be declared per-version directory rather than in the root config, enabling version-specific nav without a monolithic config file.

## ⚠️ Breaking Changes

- **Core plugins must be declared explicitly.** Auto-included plugins from 0.6.x are no longer bundled by default — add them to your `plugins` array.
- **`layout` config key renamed to `layoutSlots`.**
- **Node.js ≥ 18 is now required.**
- **Third-party plugin shorthand names are no longer resolved** — use the full package name.

## Migration Guide

Upgrade by running `npm install docmd@latest`. Then:

1. Add any previously auto-included plugins (`search`, `seo`, `sitemap`) to your `plugins` array explicitly.
2. Rename `layout` → `layoutSlots` in your config.
3. If you use third-party plugins by shorthand, update them to their full package name.
