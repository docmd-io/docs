The `docmd` 0.7.6 release introduces a **unified URL normalisation engine** that eliminates 301 redirects, enforces trailing-slash URLs for SEO, and brings consistent link handling across every surface — Markdown, buttons, tags, navigation, and menubar. It also ships targeted SPA and developer-experience fixes.

## 🔗 Unified URL Normalisation

All URL handling now routes through a single `resolveHref()` function in `@docmd/parser`. Trailing-slash enforcement, `index.md` stripping, and double-slash prevention are applied consistently across:

- Markdown links
- Button & tag containers
- Navigation & menubar configuration
- Auto-router (zero-config mode)
- Hreflang and canonical `<link>` tags

**Key improvements:**
- No more 301 redirect chains — `/configuration/overview` → `200` directly at `/configuration/overview/`
- No `/index` artefacts or `.md` extensions in any rendered link
- Hreflang alternate tags now correctly strip locale prefixes from page paths

See the [Linking & Referencing](https://docs.docmd.io/content/syntax/linking/) guide for the full URL translation table.

## 🆕 Universal `external:` and `raw:` Prefixes

### `external:` — Force New Tab

Add the `external:` prefix to any link to open it in a new tab. Works in Markdown, buttons, tags, navigation, and menubar:

```markdown
[Open in new tab](external:./configuration/overview.md)
::: button "API Docs" external:./api/node-api.md
```

**Behaviour change:** HTTP/HTTPS links no longer auto-open in a new tab. Use `external:` explicitly when you want that behaviour — this gives you full control, including over links to your own external documentation sites.

### `raw:` — Bypass Normalisation

Link to a raw `.md` file without the engine stripping its extension:

```markdown
[Download Template](raw:templates/starter.md)
```

Both prefixes are stripped from the rendered HTML — they are build-time signals only.

See the [Linking & Referencing](https://docs.docmd.io/content/syntax/linking/) guide for details.

## 🧩 Centralised URL API for Plugins

A new URL utility layer in `@docmd/parser` is re-exported via `@docmd/api`. Plugins no longer need to parse `outputPath` manually — the engine pre-computes clean URLs for every page:

```typescript
import { outputPathToSlug, sanitizeUrl } from '@docmd/api';

export async function onPostBuild({ pages }) {
  for (const page of pages) {
    page.urls.slug;      // "guide/"
    page.urls.canonical; // "https://example.com/guide/"
    page.urls.pathname;  // "/guide/"
  }
}
```

See the [Plugin Development](https://docs.docmd.io/advanced/plugins/) guide for the full API reference.

## 🔧 Developer Experience

### Silent Plugin Warnings in Dev Mode

Previously, a missing or misconfigured plugin (e.g. `@docmd/plugin-math`) would print a warning on **every file change** during `docmd dev`:

```
↻ Change in docs/en/guide.md... ⚠️  Could not load plugin: @docmd/plugin-math
Done.
```

Plugin warnings are now **printed once** per dev session and suppressed on subsequent rebuilds. The first warning still appears so you know something is wrong — it just doesn't flood the terminal.

### SPA Hash Navigation Fixed

In SPA mode, clicking a `#heading` anchor on the current page no longer silently fails. Previously, after a `pushState` navigation, the browser's native scroll-to-anchor did not fire. The SPA router now intercepts same-page hash clicks and scrolls to the target element smoothly:

```markdown
[Jump to Configuration](#configuration)  <!-- works in SPA mode now -->
```

Browser back/forward through hash history (via `popstate`) also scrolls correctly.

## 🚀 SEO Improvements

- All internal URLs rendered with trailing slashes — eliminates 301 redirect chains
- Canonical and hreflang `<link>` tags use correct clean paths
- Zero `.md` extensions or `/index` artefacts in production HTML

## Migration Guide

Update with `npm update @docmd/core` or `pnpm update @docmd/core`. No configuration changes required.

::: callout info
If you have custom server-side redirect rules that strip trailing slashes, remove them. The engine now produces the canonical trailing-slash format natively.
:::
