The `docmd` 0.7.6 release introduces a **unified URL normalisation engine** that eliminates 301 redirects, enforces trailing-slash URLs for SEO, and brings consistent link handling across every surface.

## ✨ Highlights

### 🔗 Unified URL Normalisation

All URL handling now routes through a single `resolveHref()` function in `@docmd/parser`. This ensures that trailing-slash enforcement, `index.md` stripping, and double-slash prevention are applied identically across Markdown links, button containers, and navigation menus.

- **Eliminate 301 Redirects**: URLs like `/guide/overview` now resolve directly to `/guide/overview/` at build time, saving a network round-trip.
- **Clean Path Resolution**: No more `/index` artefacts or `.md` extensions in any rendered link.
- **Global Consistency**: The same logic applies to auto-routers, hreflang tags, and canonical links.

### 🆕 Universal `external:` and `raw:` Prefixes

We've introduced two new magic prefixes to give you total control over link behaviour across all `docmd` components.

- **`external:` — Force New Tab**: Add this to any link (Markdown, Buttons, Menubar) to force it to open in a new tab.
- **`raw:` — Bypass Normalisation**: Use this when you need to link to a file (like `template.md`) without the engine stripping its extension.

### 🧩 Centralised URL API for Plugins

Plugins no longer need to parse `outputPath` manually. We've exposed a new URL utility layer via `@docmd/api` that provides pre-computed, clean URLs for every page during the build process.

## 🐛 Bug Fixes & UX

### ⚡ SPA Hash Navigation Fix
In SPA mode, clicking a `#heading` anchor on the current page no longer silently fails. The SPA router now intercepts same-page hash clicks and scrolls to the target element smoothly.

### 🔕 Silent Plugin Warnings
Missing or misconfigured plugins now only print a warning **once per dev session**. Subsequent rebuilds in `docmd dev` will suppress these warnings to keep your terminal clean while you work.

### 🚀 SEO Improvements
- **Trailing Slashes**: All internal URLs are now rendered with trailing slashes by default to follow SEO best practices.
- **Canonical Clarity**: Canonical and hreflang `<link>` tags now use correct, normalized paths automatically.

## Migration Guide

Update to the latest version with `npm update @docmd/core` or `pnpm update @docmd/core`. 

::: callout tip "Server Configuration"
If you have custom server-side redirect rules that strip trailing slashes, you should remove them, as the engine now produces the canonical trailing-slash format natively.
:::
