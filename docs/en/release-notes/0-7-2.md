The `docmd` 0.7.2 release introduces the highly anticipated **`docmd deploy`** command. This new utility is designed to bridge the gap between "building" your docs and "hosting" them in professional, production-grade environments.

## ✨ Highlights

### 🛡️ Introducing the 'deploy' Command

For the first time, `docmd` provides built-in scaffolding for production hosting. Instead of manually writing server configs, you can now generate optimized, security-hardened templates instantly:
- **Docker**: Optimized multi-stage builds with precise layer caching and exact version pinning for 100% reproducible builds.
- **NGINX & Caddy**: Production-ready configurations that include industry-standard security headers (`X-Content-Type-Options: nosniff`, `X-Frame-Options: SAMEORIGIN`) by default.
- **Performance Out-of-the-Box**: Automated GZIP compression and aggressive "immutable" caching policies for static assets.
- **SPA Resilience**: Intelligent 404 handling and SPA routing fallbacks pre-configured for every target.

### ⚠️ Safety First & Idempotency

Even though this is a new feature, we've built it with maturity in mind. The `deploy` command is **idempotent**, meaning it will never accidentally overwrite your files. Use the new `--force` flag when you explicitly want to regenerate your configurations from our latest templates.

### 🛡️ The 7-Pillar Failsafe

To ensure this release is rock-solid, our internal verification engine has been refactored into **7 logical pillars of stability**. This includes a new **Dynamic Integrity Engine** that captures version mismatch regressions instantly across the entire monorepo.

## 📝 Complete Changelog

### 🚀 Features & Enhancements
- **New Command**: Introduced `docmd deploy` with support for `--docker`, `--nginx`, `--caddy`, and `--force`.
- **Docker Scaffolding**: Automated Dockerfile generation with `package.json` layer caching and core-version pinning.
- **Web Server Scaffolding**: Hardened Nginx and Caddy templates with security headers, GZIP, and intelligent 404/SPA routing.
- **Failsafe**: Consolidated verification logic into a faster, more professional 7-stage pipeline.
- **Versioning**: New integrity engine automatically flags outdated internal workspace references.
- **SEO Plugin**: Fully respects the `titleAppend` frontmatter property for social media (OG/Twitter) metadata.

### 🐛 Bug Fixes
- **Caddy Config**: Corrected Cache-Control header syntax (quoted values) for Caddyfile compatibility.
- **Bump Script**: Fixed a regex `lastIndex` bug in `scripts/bump.js` that caused inconsistent version replacements.
- **CLI Safety**: Added proper async error handling to all deployment command entry points.
- **Live Editor**: Fixed mobile overflow issues and template switching state tracking.

## Migration Guide

For **end users**: No breaking changes. Simply update and try the new `docmd deploy` command to see how easily you can pack your site for production.

For **plugin authors**: No changes required.
