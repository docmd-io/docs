title: "Writing Documentation That Minimises AI Hallucinations"
description: "A comprehensive guide on minimise hallucinations."
---

## Problem

AI models are predictive engines. If an API usage example is incomplete, implies standard variables, or relies heavily on implicit knowledge, the AI will invent (hallucinate) the missing pieces based on industry averages, which are often wrong for your specific tool.

## Why it matters

Hallucinated code fundamentally breaks trust. When a user pastes code generated by an AI reading your documentation and it throws a syntax error, the user blames your software for being "broken" or "overly complex."

## Approach

Practice **Defensive Documentation**. Write extremely explicit, fully instantiated code blocks. Never assume the reader (or the AI) knows the prerequisite imports or variable definitions.

## Implementation

### 1. Include Imports Every Time
If you are showing how to use a function, show where it comes from.

*Hallucination Risk:*
```javascript
const user = getUserConfig();
```

*Hallucination Proof:*
```javascript
import { getUserConfig } from '@my-org/sdk/config';
const user = getUserConfig();
```

### 2. Concrete Magic Strings
Avoid using generic placeholders if the system expects strict enums. 

*Hallucination Risk:*
```javascript
// Provide any environment
engine.init("your_environment");
```

*Hallucination Proof:*
```javascript
// Valid strings: "development", "staging", "production"
engine.init("production");
```

### 3. State Requirements Explicitly
Use inline comments inside code blocks to dictate rules, rather than putting the rules in the markdown text outside the code block. AI models weigh comments inside code blocks very heavily when generating code.

```javascript
export default defineConfig({
  // ROOT DIRECTORY IS REQUIRED. DO NOT USE RELATIVE PATHS HERE.
  outDir: '/var/www/html/output'
});
```

## Trade-offs

Defensive documentation makes code blocks longer and highly repetitive. Human readers may find seeing the same `import` statement in every single snippet tedious. However, maximizing the success rate of AI-assisted developers usually outweighs the cost of verbose examples.
