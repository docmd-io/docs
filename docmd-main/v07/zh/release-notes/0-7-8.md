`docmd` 0.7.8 版本引入了作为核心功能的 **Git 插件**用于仓库感知元数据、官方插件的**自动安装**、为从其他文档引擎迁移的用户提供的**容器语法兼容性**，以及平台上的各种生活质量改进。

## 亮点

### Git 插件（核心）

新的 **Git 插件**现已包含在 `@docmd/core` 中，默认启用。它为您的文档页面带来仓库感知的元数据 - 最后更新时间戳、提交历史工具提示和编辑链接 - 所有这些都直接从您的 Git 历史记录中获取，无需配置。

```javascript
// 可选：配置编辑链接
plugins: {
  git: {
    repo: 'https://github.com/your-org/docs',
    branch: 'main'
  }
}
```

**功能：**
- **最后更新时间戳**：显示每个页面的最后修改时间，对于最近的更改使用相对格式
- **提交历史工具提示**：悬停可查看带有作者名称和消息的最近提交
- **编辑链接**：自动为 GitHub、GitLab 和 Bitbucket 构建"编辑此页"链接
- **优雅降级**：如果项目不在 Git 仓库中，会自动禁用

该插件用功能更丰富的替代方案取代了旧的 `editLink` 配置选项。有关完整详情，请参阅 [Git 插件文档](/plugins/git)。

### 官方插件自动安装

在您的 `docmd.config.js` 中列出的官方插件现在会在缺失时自动安装。当您向配置中添加未安装的插件时，docmd 会在下次构建时从 npm 下载它。

```javascript
// docmd.config.js
plugins: {
  pwa: {},     // 未安装？docmd 自动安装
  threads: {}  // 同样
}
```

**安全特性：**
- 仅适用于注册表中的官方 `@docmd/plugin-*` 包
- 安装与您的 `@docmd/core` 版本匹配的确切版本
- 使用您项目的包管理器（npm、pnpm、yarn 或 bun）
- 在终端显示进度

### 容器语法兼容性

从 **VitePress**、**Docusaurus** 或类似文档引擎迁移的用户现在可以使用熟悉的容器语法而无需修改。

**VitePress 风格的容器现在可以工作：**
```markdown
:::tip
这将渲染为提示提示框。
:::

:::warning
这将渲染为警告提示框。
:::

:::details 点击展开
这将渲染为可折叠部分。
:::
```

**Docusaurus 风格的容器现在可以工作：**
```markdown
:::note
这将渲染为信息提示框。
:::

:::caution
这将渲染为警告提示框。
:::
```

此外，所有容器现在都支持**无空格语法**：
```markdown
:::callout info
与 ::: callout info 效果相同
:::

:::tabs
与 ::: tabs 效果相同
:::
```

这允许从其他引擎迁移文档时只需最少的更改。

### 迁移友好的别名

| 别名 | 映射到 | 来源 |
| :--- | :--- | :--- |
| `:::tip` | `callout tip` | VitePress |
| `:::warning` | `callout warning` | VitePress |
| `:::danger` | `callout danger` | VitePress |
| `:::info` | `callout info` | VitePress |
| `:::details` | `collapsible` | VitePress |
| `:::note` | `callout info` | Docusaurus |
| `:::caution` | `callout warning` | Docusaurus |

::: callout tip "无缝迁移"
这些别名与标准 `docmd` 语法一起静默工作。您现有的文档继续正常工作，而从其他引擎导入的内容也能正确渲染。
:::

## 内部改进

### 英式英语标准化

文档已更新为在整个过程中一致使用英式英语拼写。这包括在适当的地方使用 "optimised"、"centralised"、"localisation" 和 "colour" 等术语。

### 代码质量

- 在文档和代码注释中用标准连字符替换了长破折号，以提高可读性
- 标准化源文件中的注释格式
- 改进了插件 API 的 TypeScript 类型定义

## 完整更新日志

### 功能
- **Git 插件**：用于最后更新时间戳、提交历史和编辑链接的新可选插件
- **容器别名**：VitePress 和 Docusaurus 容器语法现在可以开箱即用
- **无空格容器**：所有容器现在接受 `:::` 后有或没有空格的语法

### 改进
- **文档**：英式英语拼写标准化
- **代码风格**：整个代码库的一致格式
- **插件注册表**：将 Git 插件添加到官方注册表

### 弃用
- **editLink 配置**：独立的 `editLink` 配置选项已弃用，推荐使用 Git 插件

## 迁移指南

### 采用 Git 插件

如果您之前使用 `editLink` 配置：

**之前：**
```javascript
export default defineConfig({
  editLink: {
    enabled: true,
    baseUrl: 'https://github.com/org/repo/edit/main/docs'
  }
});
```

**之后：**
```javascript
export default defineConfig({
  plugins: {
    git: {
      repo: 'https://github.com/org/repo',
      branch: 'main'
    }
  }
});
```

### 对于从其他引擎迁移的用户

如果您从 VitePress、Docusaurus 或类似引擎导入文档：

1. 您现有的 `:::tip`、`:::warning`、`:::note` 容器将正确渲染
2. 无空格语法如 `:::tabs` 与标准 `::: tabs` 一起工作
3. 无需更改您的 markdown 文件

我们建议逐步过渡到标准 `docmd` 语法（`::: callout tip`）用于新内容，因为它提供了更多标题和图标的灵活性。
