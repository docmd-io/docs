`docmd` 0.7.7 版本引入了**多项目支持** —— 能够从单个 docmd 实例构建和提供多个独立的文档站点。一条命令，一个域名，多个独立项目。

## ✨ 亮点

### 多项目配置

仓库根目录下的 `docmd.config.js` 现在可以定义多个文档项目。每个项目拥有独立的前缀、配置、版本管理和导航 —— 同时共享相同的主题、资源和部署流程。

```javascript
// docmd.config.js（根配置）
module.exports = defineConfig({
  projects: [
    { prefix: '/',       src: 'docmd-main' },
    { prefix: '/search', src: 'docmd-search' }
  ]
});
```

**输出结构：**
```
site/                     ← 根项目 (docmd-main)
site/search/              ← 子路径项目 (docmd-search)
```

每个项目文件夹包含自己的 `docmd.config.js`，具有完全独立的配置 —— 标题、国际化、版本、插件、导航、布局等一切设置。

详见[多项目配置](/configuration/multi-project/)指南。

### 共享资源

放置在仓库根目录 `assets/` 中的资源会自动流向所有项目。每个项目也可以拥有自己的 `assets/` 目录，优先级高于共享资源。

### 独立版本与国际化

每个项目维护独立的版本历史和国际化设置。文档引擎 v0.7 和搜索工具 v0.1 可以在同一域名下共存，不会产生版本冲突。

### 多项目开发服务器

多项目模式下的 `docmd dev` 会构建所有项目并通过单个端口提供服务。任何项目中的文件更改都会触发定向重建和实时刷新。

```
────────────────────────────────────────
  多项目开发服务器

  本地：    http://127.0.0.1:3000
  网络：    http://192.168.1.5:3000

  项目：  http://127.0.0.1:3000/ → project-alpha/
  项目：  http://127.0.0.1:3000/search → project-beta/
────────────────────────────────────────
```

## 🔧 架构

多项目处理器位于 `engine/projects.ts` —— 一个独立模块，不与其他核心文件混合。它：

1. 在正常构建开始前**检测**根配置中的 `projects[]`
2. **验证**前缀（无重复，必须有根 `/`）
3. **遍历**每个项目，切换到其目录
4. **委托**标准 `buildSite()` 处理
5. **合并**所有输出到单个 `site/` 目录

对现有构建流程零影响。标准单项目工作流继续完全按原样运行。

## 🚀 部署

多项目输出为单个静态 `site/` 目录 —— 兼容 GitHub Pages、Netlify、Vercel、Cloudflare Pages 及所有静态托管服务。无需 nginx 代理配置。

## 升级指南

**现有单项目配置无需更改。** 多项目模式仅在根配置中存在 `projects` 数组时激活。

::: callout info
根 `docmd.config.js` 应仅包含 `projects` 数组。所有其他配置（标题、主题、插件等）属于各项目自己的 `docmd.config.js`。
:::
