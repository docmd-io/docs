## 🐛 Fehlerbehebungen

### Plugin-Auto-Installation: robust gegen `import`-only `exports`

Der Auto-Installer in `@docmd/api` löste installierte Pakete zuvor mit
Node-CommonJS-`require.resolve` auf und lud die aufgelöste Pfad-URL
anschließend mit `await import(file://path)`. Bei Paketen, die im
`exports`-Feld nur eine `import`-Bedingung deklarieren, wirft
`require.resolve` den Fehler `ERR_PACKAGE_PATH_NOT_EXPORTED`, und der
Build gibt die Meldung

```
⬢ Could not load @docmd/template-summer after auto-install
```

aus, obwohl das Paket korrekt installiert ist. Der Retry-Pfad verwendet
nun direkt `await import(name)`, das das `exports`-Feld nativ
berücksichtigt und mit jeder Bedingung funktioniert.

Im Retry-Pfad findet zusätzlich eine Registry-Reprüfung statt
(Defense-in-Depth). Die Menge der Namen, die automatisch installiert
werden können, bleibt unverändert – nur die Menge der
`exports`-Bedingungen, die sie tragen dürfen, ist nun größer.

### `@docmd/template-summer@0.8.9`: `default`-Exports-Bedingung hinzugefügt

`@docmd/template-summer` wurde mit einem `exports["."]` ausgeliefert, das
nur die `import`-Bedingung deklarierte. Der erste `require.resolve`-
Versuch im Loader schlug deshalb fehl, obwohl das Paket korrekt
installiert war. Das Paket deklariert nun:

```json
"exports": {
  ".": {
    "types": "./dist/index.d.ts",
    "import": "./dist/index.js",
    "default": "./dist/index.js"
  }
}
```

Dies entspricht der Konvention von `@docmd/plugin-git`,
`@docmd/plugin-openapi` und `@docmd/plugin-threads`. Aktualisieren Sie
Ihre Lockfile-Datei (`npm install`, `pnpm install`, `yarn install` oder
`bun install`), nachdem Sie `@docmd/core` auf `0.8.9` aktualisiert haben.

### `docmd-search@0.1.0`: stabil, nicht mehr Alpha, mit `default`-Exports

`docmd-search` ist nicht mehr mit `alpha` getaggt. Die Version `0.1.0`:

- Lässt den Suffix `-alpha.1` weg. Die Version auf dem `latest`-Tag ist
  nun `0.1.0`. Die beiden `0.1.0-alpha.*`-Versionen sind veraltet – `npm
  install docmd-search` löst korrekt zu `0.1.0` auf.
- Fügt `"default": "./dist/index.js"` sowohl zum `.`- als auch zum
  `./client`-Subpfad in `exports` hinzu.
- Entfernt das Flag `--tag alpha` aus dem Publish-Workflow. Alle
  zukünftigen Releases landen auf dem Standard-`latest`-Tag.

Wenn Sie in Ihrer Doku semantische Suche nutzen
(`plugins.search.semantic = true`), aktualisieren Sie Ihre
`node_modules`:

```bash
rm -rf node_modules/docmd-search
npm install         # bzw. pnpm/yarn/bun install
```

### Plugin-Loader: gehärtete `resolveDocmdSearch`-Fallback-Kette

Der Resolver des Search-Plugins für die optionale `docmd-search`-
Peer-Abhängigkeit verwendet beim Lesen des Manifests nun eine
explizite `import` → `default` → `require` → `main`-Fallback-Kette sowie
eine manuelle `node_modules`-Suche als Rückfallebene. Die Funktion ist
damit robust gegenüber gängigen Installations-Layout-Varianten, ohne
dass sich ihr Rückgabewert ändert.

## 🔒 Sicherheit

Keine neue Angriffsoberfläche. Die Vertrauensgrenze für die automatische
Installation bleibt die offizielle Plugin-Registry-Allowlist plus die
Präfix-Prüfung `@docmd/plugin-*` / `@docmd/template-*`. Der neue
`import(name)`-Retry-Pfad erweitert nicht die Menge der installierbaren
Pakete – er erweitert nur die Menge der `exports`-Bedingungen, die ein
registry-zugelassenes Paket tragen darf.

## 📚 Dokumentations-Updates

- **Plugins → Plugins verwenden** – der Abschnitt „Auto-Installation"
  erwähnt nun den robusten Resolver und die Registry-Reprüfung.
- **Plugins → Search** – der Abschnitt „Automatic Installation" erwähnt
  die gehärtete `resolveDocmdSearch`-Fallback-Kette.
- **Entwicklung → Plugins entwickeln** und **Entwicklung → Templates
  entwickeln** – neuer Unterabschnitt „ESM-Exports – die
  `default`-Bedingung".
