### ✨ Highlights

Dieses Release behebt einen kritischen Rendering-Fehler im Live-Editor-Dienst, führt einen `migrate --upgrade`-Befehl für die automatisierte Konfigurationsmodernisierung ein und poliert die Projekt-Switcher-UI.

### Live-Editor-Fehlerbehebung

Ein Fehler beim Template-Rendering, der den Live-Editor-Dienst (`live.docmd.io`) betraf, wurde behoben.

Die Variable `workspace` wurde in `layout.ejs` aufgerufen, bevor sie im Browser-Rendering-Kontext definiert wurde, was bei jedem Seiten-Render zu folgendem Fehler führte:

```
Error: Template Render Error: Cannot access 'workspace' before initialization.
```

Der Browser-Einstiegspunkt übergibt nun ordnungsgemäß `workspace: null` an den Template-Renderer und behebt den Crash für alle Benutzer der Live-Preview-Umgebung.

### Konfigurationsupgrade-Befehl

Der Befehl `docmd migrate` erhält ein neues `--upgrade`-Flag. Das Ausführen schreibt Ihre Konfigurationsdatei automatisch in das moderne Schema um.

```bash
npx @docmd/core migrate --upgrade
```

Das Upgrade behandelt automatisch die folgenden Legacy-Schlüsselmappings:

| Legacy-Schlüssel | Moderner Schlüssel |
| :--- | :--- |
| `projects` (auf oberster Ebene) | `workspace.projects` |
| `siteTitle` | `title` |
| `siteUrl` / `baseUrl` | `url` |
| `srcDir` / `source` | `src` |
| `outputDir` | `out` |
| `defaultLocale` | `i18n.default` |

Die ursprüngliche Datei wird an Ort und Stelle neu geschrieben. Ein Backup ist nicht erforderlich, da die Migration nicht destruktiv ist - es werden nur Schlüssel umbenannt und ihre Werte nicht geändert.

### TOC-HTML-Entitätsdekodierung

Überschriftentexte mit Sonderzeichen (wie `<`, `>`, `&` oder Smart Quotes) werden jetzt in der Inhaltsverzeichnis-Seitenleiste korrekt dekodiert. Zuvor wurden Überschriften mit diesen Zeichen als rohe HTML-Entitäten (z. B. `&amp;`) anstelle der beabsichtigten Zeichen angezeigt.

### Änderungsprotokoll

1. **Live-Editor**: Fehler beim Template-Rendering behoben, verursacht durch den Zugriff auf die Variable `workspace`, bevor sie im Browser-Rendering-Kontext initialisiert wird.
2. **Konfigurationsupgrade**: `docmd migrate --upgrade` hinzugefügt, um Legacy-Konfigurationsschlüssel automatisch ins moderne Schema zu schreiben.
3. **TOC-Rendering**: HTML-Entitätsdekodierung im Inhaltsverzeichnis-Überschriftentexte behoben, damit Sonderzeichen korrekt angezeigt werden.
4. **UI-Poliert**: Projekt-Switcher im Optionsmenü gestylt, um visuell mit i18n- und Erscheinungsschaltflächen übereinzustimmen, und Schaltflächenreihenfolge korrigiert.
