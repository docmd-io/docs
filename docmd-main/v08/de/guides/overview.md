# Leitfäden-Übersicht

Leitfäden sind fortgeschrittene Inhalte zum Thema "wie mache ich X gut". Wählen Sie den Cluster, der zu Ihrem Ziel passt. Die meisten Benutzer beginnen mit **Schreibqualität** — diese Leitfäden sind für jede docmd-Site relevant, unabhängig vom Stack oder Funktionsfokus.

::: grids
::: grid
::: card "Schreibqualität" icon:pen-tool
Für jeden docmd-Benutzer. Skalierbares technisches Schreiben, Lesbarkeit, Aufgabe vs. Konzept und das Vermeiden von Anti-Patterns. Der universell relevanteste Cluster.
[Mit Schreibqualität starten →](./writing-quality/scalable-technical-writing)
:::
:::
::: grid
::: card "Workflows & Teams" icon:users
Für Dokumentations-Verantwortliche und Teams. Git-basierte Workflows, Vorschauumgebungen, Konsistenz im großen Stil und Release-Workflows.
[Workflows & Teams erkunden →](./workflows-teams/setting-up-workflow)
:::
:::
::: grid
::: card "Suche" icon:search
Für jeden mit Suchfeld. Relevanz verbessern, Geschwindigkeit optimieren, Local-First-Indexierung und semantische Suche hinzufügen.
[Suche optimieren →](./search/improving-search-relevance)
:::
:::
::: grid
::: card "Performance" icon:gauge
Für Teams, denen Time to Interactive wichtig ist. Sub-100ms-Navigation, CDN-Deployment, JS-Payload-Reduktion, Low-End-Geräte, Caching.
[Ihre Site beschleunigen →](./performance-delivery/sub-100ms-navigation)
:::
:::
::: grid
::: card "KI & LLMs" icon:brain-circuit
Für Teams, deren Dokumentation auch für AI-Agenten und LLMs lesbar sein soll. MCP, Agent Skills, llms.txt, Semantic-Search-Design, Chunking, Kontext-Erhaltung.
[Dokumentation AI-bereit machen →](./ai-optimisation/mcp-and-agent-skills)
:::
:::
::: grid
::: card "Integrationen" icon:plug
Für die Anbindung von docmd an Ihren restlichen Stack. OpenAPI-Generierung, GitHub Actions, bestehende Markdown-Repos, parallele Tools.
[Integrationen ansehen →](./integrations/choosing-deployment-method)
:::
:::
:::

::: callout tip "Nicht sicher, wo Sie anfangen sollen?"
Wenn Sie brandneu bei docmd sind, überspringen Sie die Leitfäden und lesen Sie zuerst [Erste Schritte](/getting-started/quick-start). Leitfäden setzen voraus, dass Sie mit den Grundlagen vertraut sind.
:::
