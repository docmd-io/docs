### ✨ Highlights

This release ships the **Custom Template Plugin API** — a fully modular way to ship alternative layouts as standalone plugins — showcased by **Summer**, the first official community template (installable via a new `docmd add summer` command). Two opt-in **core UI features** (cookie consent dialog and site-wide announcement banner) are now built into `@docmd/ui` so every template can adopt them without a separate plugin. Also includes a new asset priority system, **Docker `:latest` tag fix** (quick-start now works out of the box), and zero breaking changes.

## 🧩 Custom Template Plugin API

Templates are now plugins. A template package declares `capabilities: ['template']` and ships a `templates[]` array of `.ejs` file overrides. The resolver in `@docmd/ui` (new `resolveTemplate()`) honours this resolution chain:

1. `frontmatter.template` (per-page)
2. `config.templates[glob]` (per-section, future)
3. `config.theme.template` (site-wide)
4. Built-in default from `@docmd/ui`

If any step fails or the resolved file is missing on disk, the resolver falls back to the default and emits a single TUI warning. Templates ship their own CSS/JS via `templateAssets[]`, which are loaded at priority `10` so user `customCss` (priority `15`) always wins. Templates **must not** use `!important` in CSS.

**Resolution example**

```js
// template-summer/index.js
export default {
  plugin: {
    name: 'template-summer',
    version: '0.1.0',
    capabilities: ['template'],
  },
  templates: [
    { type: 'layout', templatePath: '...' },
    { type: 'sidebar', templatePath: '...' },
  ],
  templateAssets: [
    { type: 'css', path: '.../summer.css', priority: 10, position: 'head' },
  ],
};
```

```json "docmd.config.json"
// docmd.config.json
{ "theme": { "template": "summer" } }
```

Templates may also omit any of the 14 supported slots (`layout`, `404`, `toc`, `navigation`, `sidebar`, `header`, `footer`, `menubar`, `options-menu`, `project-switcher`, `version-dropdown`, `language-switcher`, `banner`, `cookie-consent`); the resolver automatically falls back to the default for any slot not provided.

**Per-page override via frontmatter**

```markdown
---
title: "Changelog"
template: "template-changelog"
---
```

The first official template package — `@docmd/template-summer` (the **Summer** layout) — ships in lockstep with this release and is installable through the standard `docmd add` pipeline.

## ☀️ Summer Template Preview

The first official template built on the Custom Template API is **Summer** — a bright, hopeful, summer-feel layout that ships with a top search bar, soft halo accents, and a denser card grid. Install it through the standard `docmd add` pipeline:

```bash
docmd add summer
```

This installs `@docmd/template-summer` from npm and updates your `docmd.config.json` automatically:

```diff
  module.exports = defineConfig({
+   theme: {
+     template: "summer"
+   },
    // ... your existing config
  });
```

Unlike regular plugins, Summer writes to `theme.template` (not `plugins.*`) and **replaces** any previously-set template — there is no stacking. Run `docmd remove summer` to revert to the default. Summer ships a dedicated `404.ejs`; the default 404 is used as a fallback when the slot is omitted.

> **Release sequencing:** `@docmd/template-summer@0.8.6` is being published ahead of the rest of the monorepo to satisfy npm's OIDC first-publish requirement (which must originate from the maintainer's local machine). The version published in lockstep with the monorepo at `0.8.7` is functionally identical.

## 🍪 Cookie Consent Dialog

A minimal, accessible cookie consent dialog is now bundled with the default UI. Enable it from your config; no plugin install required.

```json "docmd.config.json"
{
  "cookie": {
    "enabled": true,
    "message": "We use cookies to ensure you get the best experience.",
    "policyUrl": "/privacy",
    "position": "bottom-right",
    "dismissible": true,
    "expiryDays": 180
  }
}
```

| Field | Default | Description |
|---|---|---|
| `enabled` | `true` (when `cookie` object is present) | Master switch. |
| `message` | Translation key `cookieMessage` | Dialog body text. |
| `acceptText` | Translation key `cookieAccept` | Accept button label. |
| `declineText` | Translation key `cookieDecline` | Decline button label. |
| `policyUrl` | `null` | Optional link to your privacy policy. |
| `position` | `bottom` | `bottom` \| `bottom-left` \| `bottom-right` \| `center` |
| `dismissible` | `true` | Show an "X" close button. |
| `expiryDays` | `180` | How long the choice is remembered in `localStorage`. |

If `config.cookie` is not set, **no dialog is rendered** — opt-in by design. A `docmd:cookie-consent` `CustomEvent` fires on the window after a choice is made so templates and plugins can react.

## 📢 Site-Wide Announcement Banner

A dismissable banner that sits above the menubar and below the page header. Configure inline markdown (`**bold**` and `` `code` ``) or pass raw `html`.

```json "docmd.config.json"
{
  "layout": {
    "banner": {
      "content": "**v0.9 ships Friday** — read the announcement.",
      "type": "info",
      "dismissible": true,
      "link": { "text": "Read more", "url": "/blog/v0-9" },
      "icon": "megaphone"
    }
  }
}
```

| Field | Default | Description |
|---|---|---|
| `content` | `""` | Inline markdown text. Mutually exclusive with `html`. |
| `html` | `""` | Raw HTML. Takes precedence over `content`. |
| `type` | `"info"` | `info` \| `success` \| `warning` \| `danger` |
| `dismissible` | `true` | Show a close (X) button. |
| `link` | `null` | `{ text, url }` for a CTA link. |
| `icon` | `null` | Lucide icon name shown on the left. |

The "dismissed" state is stored in `sessionStorage` so the banner reappears on a fresh visit. If `config.layout.banner` is not set, **no banner is rendered** — opt-in by design.

## Changelog

### New Features

- **Custom Template Plugin API** — templates are now plugins with `capabilities: ['template']`, `templates[]`, and `templateAssets[]` hooks; new `resolveTemplate()` resolver with a 4-step fallback chain (per-page → per-section → site-wide → default).
- **New types** in `@docmd/api` and `@docmd/ui`: `Capability: 'template'`, `TemplateHook`, `TemplateAssetHook`, `TemplateSlot`, `TemplateResolutionContext`, `ResolvedTemplate`, `Asset`, `AssetKind`, `AssetPosition`.
- **New registry fields** on `PluginHooks`: `templates` and `templateAssets`; new `PluginModule.templates?` / `templateAssets?` exports.
- **New resolver API** in `@docmd/ui`: `resolveTemplate()` and `clearTemplateResolverCache()`.
- **`docmd add summer`** — first-class template install that writes to `theme.template` (not `plugins.*`); `docmd remove summer` clears it.
- **Cookie consent dialog** built into `@docmd/ui` — opt-in via `config.cookie`; emits `docmd:cookie-consent` `CustomEvent`.
- **Site-wide announcement banner** built into `@docmd/ui` — opt-in via `config.layout.banner`; dismiss state stored in `sessionStorage`.
- **Asset priority system** — deterministic CSS/JS load order: base(0) → theme(5) → template(10) → customCss(15) → plugins(20).
- **14 supported template slots**: `layout`, `404`, `toc`, `navigation`, `sidebar`, `header`, `footer`, `menubar`, `options-menu`, `project-switcher`, `version-dropdown`, `language-switcher`, `banner`, `cookie-consent`.

### Bug Fixes

- **Docker `:latest` tag missing** — the published image had no `latest` tag because the workflow's `enable` expression required a `push` event on `main`, but the workflow had no `push` trigger. Pinned all docs to `:0.8.6` and reworked [`.github/workflows/docker-publish.yml`](https://github.com/docmd-io/docmd/blob/dev-0.8.7/.github/workflows/docker-publish.yml) to (a) tag `:latest` automatically on `release` events, (b) re-add a `push: branches: [main]` trigger for `:edge` rebuilds, (c) expose a manual `tag_latest` input for back-filling. See the [bug report audit](https://github.com/docmd-io/docmd/blob/dev-0.8.7/bug-report.md) for full context.
- **i18n strings not translating in partials** — `menubar`, `options-menu`, `version-dropdown`, `language-switcher`, `project-switcher`, and `toc` partials used `t('…')` calls but were never passed the `t` function from the layout (only `banner`, `footer`, and `cookie-consent` were). Added `t: typeof t === 'function' ? t : null` to every `include()` call in `layout.ejs` so all partials now translate correctly. Also added 13 missing translation keys (`close`, `cookieAccept`, `cookieConsent`, `cookieDecline`, `cookieMessage`, `cookiePolicy`, `goBack`, `lastUpdated`, `pageNotFoundMsg`, `readMore`, `recentCommits`, `searchPlaceholder`, `selectProject`) to all 7 locale files (`en`/`de`/`es`/`fr`/`hi`/`ja`/`zh`).
- **Copy code button floating above codeblock** — the button was anchored to an inner `.code-wrapper` (set via inline `style.position = 'relative'` in `docmd-main.js`) which placed it at the top-right of the `<pre>` only, ignoring the title bar above. Now the button anchors to the outer `.docmd-code-block-wrapper` (which is now `position: relative` in CSS), so it positions cleanly relative to the whole code block including its title. Also skipped creating a redundant inner wrapper for code blocks that already have one from the parser.
- **Summer template: huge space below short content** — `.summer-content` had `flex: 1 0 auto` which stretched the article to fill the grid cell, leaving a giant empty gap between the article and the footer on short pages. Removed the grow so the content takes its natural height; the footer still sticks to the bottom via the existing `margin-top: auto`.
- **Summer template: copy button missing from code-block title bar** — `summerCodeblocks()` in `summer.js` only searched for `.copy-code-button` inside `.code-wrapper`, but `injectCopyButtons()` in `docmd-main.js` appends the button directly to `.docmd-code-block-wrapper` (no inner `.code-wrapper`) for code blocks with a parser-rendered title (e.g. ```` ```js "main.ts" ````). The button stayed inline below the `<pre>` instead of moving into the title bar. `summerCodeblocks()` now searches the wrapper itself for the button, so the title bar gets the copy control in both code-block paths.
- **Summer template: changelog timeline line misaligned with entry dots** — two CSS rules for `.changelog-timeline` (one specific, one general) were partially overlapping: the specific rule set the line at `left: 0`, but the general rule added `padding-left: 32px` to the timeline container, pushing each entry 32px to the right of the line. The dot then floated 28px off the line. The specific rule's line offset is now `left: 32px`, so the line centre lines up with each entry's dot centre.
- **Docker image: quick-start crashed with "No documentation directory or configuration found"** — the image's `WORKDIR` was `/docs` and its default `CMD` was `docmd dev --host 0.0.0.0`, but `/docs` was empty in the image (the seeded template lived in `/template`). Added a small `docker-entrypoint.sh` that copies `/template` into `/docs` only when `/docs` is completely empty (so user-mounted volumes are never overwritten), then `exec`s the supplied command. The image now works out of the box with no volume mount.
- **Docker image: healthcheck always reported unhealthy** — the image's `HEALTHCHECK` hardcoded `curl http://localhost:3000/`, but `docmd dev` auto-increments to 3001+ when 3000 is taken (e.g. another dev server is already running on the host). Replaced the inline curl with `docker/docker-healthcheck.sh`, which probes 3000–3005 and returns 0 as soon as any port responds.
- **Conditional plugin asset loading broke SPA navigation** — when a page emitted a plugin's `<script src="...">` tag conditionally (e.g. Mermaid's `init-mermaid.js` only on pages with `class="mermaid"`), the script tag was rendered into `<body>` AFTER `</main>`, so SPA navigation's content swap (which only replaces `.main-content` and a fixed selector list) left the new tag sitting in the parsed `doc` but never added it to the live document. Navigating from a non-mermaid page to a mermaid page left the `.mermaid` divs unrendered (no listener, no IIFE, no init). Direct load and reload worked because the script was already in the initial HTML; KaTeX (CSS-only, in `<head>`) worked because the existing head asset sync handled it. Added a body-script sync in `navigateTo()` that mirrors the head sync: for every `body > script[src]` in the new HTML not already present in the live document, build a fresh `<script>` element (not a `cloneNode` — browsers skip cloned scripts) with the resolved absolute URL and append it. The plugin's own IIFE handles the current-page render (calls its setup directly when `document.readyState !== "loading"`); the `docmd:page-mounted` listener it registers covers subsequent navigations. The sync is fully generic — no plugin names, no keywords, no hardcoded paths — so it works for any future plugin asset.
- **Summer template: titled codeblocks lost padding and font** — the `.code-wrapper>pre, .docmd-code-block-wrapper>.code-wrapper>pre` selector only matched codeblocks with an inner `.code-wrapper` between the outer wrapper and the `<pre>`. Titled codeblocks (` ```js "title" ` `) have `<pre>` as a direct child of `.docmd-code-block-wrapper` — no inner `.code-wrapper` since `injectCopyButtons` appends the copy button directly to the wrapper. The selector missed those `<pre>`s, so they inherited zero padding and the browser-default monospace font instead of the Summer tokens (code was squished against the wrapper edges). Added `.docmd-code-block-wrapper>pre` to both the pre and pre-code selectors so all three codeblock shapes (`.code-wrapper > pre`, `.docmd-code-block-wrapper > .code-wrapper > pre`, `.docmd-code-block-wrapper > pre`) get the Summer padding (16px 18px), JetBrains Mono font (0.84rem), line-height (1.65), and fg colour.
- **Summer template: codeblock rules carried leftover `!important` that violated the template guideline** — the file's own header states "Do NOT use `!important`. User customCss (priority 15) must always win so users can restyle any selector without forking the template", but the pre/code-wrapper rules had `!important` on 8 properties. The `!important` was a leftover from when both `docmd-main.css` and `summer.css` were loaded together; in the current design `docmd-main.css` is **not** loaded by the Summer layout (per the layout's header comment), and Summer's selectors all have higher specificity than the base `pre` rule, so the `!important` was no longer needed. Removed it from the codeblock pre, code-wrapper, and pre-code rules. Empirically verified: Summer's styles still apply (padding, font, transparent background), and user `customCss` can now correctly override Summer without needing `!important` (as the guideline intended) — while `customCss` with `!important` still wins (as it should).
- **Sidebar nav expand/collapse silently broken on real links** — `navigation.ejs` puts `.nav-group` on both the dummy `<span class="nav-label">` and the real `<a class="nav-group">` (one with an `href`). The global click handler only matched `.nav-label, .collapse-icon-wrapper`, so clicking the label text of a real link (e.g. "Configuration") silently did nothing — only the chevron worked. Updated the toggle handler to also match `.nav-group`, so clicking the label text expands/collapses the group. To stop the toggle handler AND the SPA router from both firing (which would just navigate away and reset sidebar state on the new page), the SPA router now skips `<a class="nav-group">` inside `li.collapsible`. Verified on both playground (Summer template) and docs site (default theme): collapsed → click `<a>` → group expands, no SPA navigation; expanded → click `<a>` → group collapses (aria-expanded=false, submenu animates closed); leaf links (no `.nav-group`) still SPA-navigate as before.
- Asset pipeline now uses a stable priority sort, eliminating non-deterministic CSS load order when multiple plugins emitted styles.
- `loadPlugins()` now resets `templates` and `templateAssets` arrays between builds, mirroring the lifecycle of every other hook array.
- `template-resolver.ts` lints plugin names with `@docmd/` and `template-` prefixes interchangeably (`summer`, `template-summer`, `@docmd/template-summer` all resolve identically).
- New `Asset` interface type-checks the previously `any[]` return value of `getAssets()`.

### Improvements

- **Default config values** — the following keys now have sensible defaults in the config schema so a brand-new `docmd.config.json` (or no config at all) gives a fully-featured site out of the box. Override any with `false` (or the appropriate value) to opt out.

  | Key | Default | Notes |
  |---|---|---|
  | `pageNavigation` | `true` | prev/next links at the bottom of the article |
  | `copyCode` | `true` | copy-code buttons on `<pre>` blocks |
  | `autoTitleFromH1` | `true` | use the first H1 as the page title when frontmatter is absent |
  | `layout.spa` | `true` | SPA navigation between pages |
  | `layout.breadcrumbs` | `true` | breadcrumb row above the page header |
  | `layout.header.enabled` | `true` | top page header |
  | `layout.sidebar.collapsible` | `true` | sidebar can collapse on desktop |
  | `layout.sidebar.defaultCollapsed` | `false` | sidebar starts expanded |
  | `layout.optionsMenu.position` | `'header'` | options menu (search / theme switch / sponsor) goes in the header |
  | `layout.optionsMenu.components.search` | `true` | search trigger in the menu |
  | `layout.optionsMenu.components.themeSwitch` | `true` | light/dark toggle in the menu |
  | `layout.optionsMenu.components.sponsor` | `null` | opt-in (set to a URL to enable) |
  | `layout.footer.style` | `'minimal'` | one-line footer bar |
  | `layout.footer.copyright` | `` `© ${new Date().getFullYear()}` `` | auto-generated current-year copyright in the footer |
  | `layout.footer.branding` | `true` | show "Built with docmd" by default |
  | `theme.name` | `'default'` | base CSS theme |
  | `theme.appearance` | `'system'` | default colour mode (follows OS `prefers-color-scheme`). Set to `'light'` or `'dark'` to force. |
  | `theme.codeHighlight` | `true` | syntax highlighting on `<pre>` blocks |
- **Sticky-bottom layout** — `.page-navigation` (prev/next) and `.page-footer-actions` now sit at the bottom of the content area via flex `margin-top: auto`, so they no longer float in the middle on short pages. Pinned to the bottom of the article, just above the page footer.
- **Thinner header & menubar** — default header height and menubar height both reduced from 52–54px to 44px. Page-header padding trimmed from `var(--space-3)` to `var(--space-2) var(--space-3)`. Less vertical chrome, more content.
- **Smaller permalink icon** — the `heading-anchor` SVG is now `0.85em` (was `1.1em`). Subtler by default; still reveals on heading hover.
- **Polished default 404 page** — the built-in `404.ejs` now ships a glass card layout with a soft halo background (tied to `--link-color` / `--glow-color`), a large gradient-feel "404" headline, and **Return home** + **Go back** action buttons (each with a Lucide icon). Mirrors the Summer template's 404 design language using only default theme tokens, so it works with any `theme.name`. The `404` glyph itself is rendered through the `errorCode404` translation key for screen-reader consistency.
- **404 page fully translated** — every user-visible string on the default 404 page (`errorCode404`, `pageNotFound`, `pageNotFoundMsg`, `returnHome`, `goBack`) is now wrapped in `t()` with English fallbacks, and the `<html lang="…">` + `dir="…"` attributes are derived from `locals.activeLocale` so the page renders correctly in RTL locales (e.g. Arabic, Hebrew when added). The same keys are already present in all 7 locales.
- **Card title areas shrunk** — the `.card .card-title` block now uses `0.95rem` font (down from `1.125rem`), `var(--space-2) var(--space-4)` padding (down from `var(--space-3) var(--space-6)`), and `var(--space-3)` bottom margin (down from `var(--space-4)`). Denser feature grids; cards feel less top-heavy when stacked in a `grids` container.
- **Three new translation keys** — `poweredBy` ("Powered by" / localized), `viewSource` ("View source" / localized), and `goToTop` ("Go to top" / localized) added to all 7 locale files. Use them in templates or via `i18n.locales[].translations` overrides in `docmd.config.json`.
- **General UI modernisation** — added 4 halo tokens (`--glow-color`, `--glow-color-strong`, `--halo-soft`, `--halo-soft-strong`) and applied them consistently to cards, buttons, focus rings, the active sidebar pill, the step number, the 404 page, and the hero. Card hover now combines a softer shadow + halo + `color-mix()` border tint. A unified `0.15s ease` transition was added to common interactive elements. `:focus-visible` now gains a halo glow so keyboard focus is more obvious.
- **Text & colour contrast bumps** — light `--text-color` `#3f3f46` → `#27272a`; dark `--text-color` `#cacaca` → `#d4d4d8`; dark `--text-muted` `#71717a` → `#94949e`; dark `--bg-color` `#111` → `#0d0d0f`; dark `--link-color` `#068ad5` → `#38bdf8`. All values pass WCAG AA against their respective background colours.
- **Active sidebar link redesigned** — replaced the `inset 2px 0` left-border with a 3px "node on thread" pill at the link's left edge, centred vertically, overlapping the existing `.submenu` thread border. Reads as a clear active marker without competing with the sidebar's divider lines.
- **Step permalinks** — every step `<li>` now has a globally-unique `id="step-N"` and a hidden `.step-permalink` anchor (Lucide `link-2` icon) that fades in on hover/focus. `scroll-margin-top` lands below the sticky header so anchor jumps don't hide the step heading.
- `theme.name` (CSS themes) keep working and stack correctly with templates.
- `customCss` / `customJs` arrays still load last.
- `frontmatter.template` previously resolved against the old `templates` lookup table — that lookup is preserved for any keys still present (`docmd-search`, `no-style`); new keys now route through the resolver.
- `frontmatter.noStyle` continues to use the default `no-style` template.
- All hooks registered prior to 0.8.7 still fire in the same order.
- Existing `Asset` aliases (`src` / `dest` / `location` / `attributes`) preserved as deprecated for plugin back-compat.
