## 🚀 Release Highlight: Auto-Deploy V2

The deployer has been extracted into a dedicated `@docmd/deployer` package with a modular per-provider architecture. Three new cloud targets are now available:

```bash
docmd deploy --github-pages   # generates .github/workflows/deploy.yml
docmd deploy --vercel          # generates vercel.json
docmd deploy --netlify         # generates netlify.toml
```

All generated files are personalised from your `docmd.config.json` — output directory, site URL, SPA routing, and Node version are injected automatically. The existing `--docker`, `--nginx`, and `--caddy` targets continue to work unchanged.

## Changelog

#### 🔒 Security
- **Installer**: Mitigated potential command injection and ReDoS vulnerabilities by strictly enforcing the official registry during plugin installation, replacing `execSync` with `execFileSync`, and escaping configuration keys.
- **Threads Plugin**: Hardened thread comment markup by rendering through a scoped path with raw HTML disabled, whilst preserving standard markdown formatting (#136).

#### 🐛 Bug Fixes & Improvements
- **Core Engine**: The build engine now forcefully exits with a non-zero code if any plugin throws an error during build lifecycle hooks, preventing silent failures in CI/CD pipelines (#134).
- **Plugin API**: Added an optional `filePath` argument to the `onBeforeParse` and `onAfterParse` hooks, allowing developers to identify precisely which file triggered an operation or error (#135).
- **Dev Server**: Fixed an issue where switching between projects could serve a stale cached page without reflecting the correct sidebar and navigation. HTML responses now include `Cache-Control: no-cache`.

#### 🚀 Features
- **Markdown Configuration**: Introduced a new `markdown.breaks` configuration option in `docmd.config.json`. You can now set this to `false` to disable the default markdown-it line break behaviour and maintain 80-column text reflows (#137).

```json
{
  "markdown": {
    "breaks": false
  }
}
```
