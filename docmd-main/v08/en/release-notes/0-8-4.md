This release focuses on deployment improvements, safer plugin behaviour, and stability fixes across the development server and UI.

## ✨ Highlights

### Deployment targets

The deployment system has been moved into a dedicated `@docmd/deployer` package with provider-based deployment targets.

You can now generate deployment files directly for GitHub Pages, Vercel, and Netlify:

```bash id="8v6sda"
docmd deploy --github-pages
docmd deploy --vercel
docmd deploy --netlify
```

Generated files automatically use values from your `docmd.config.json`, including the output directory, site URL, SPA routing, and Node.js version.

Existing deployment targets for Docker, Nginx, and Caddy continue to work unchanged.

### Markdown line breaks

A new `markdown.breaks` option has been added to `docmd.config.json`.

Set this to `false` to disable automatic markdown line breaks and preserve wrapped markdown formatting.

```json id="9wnr4m"
{
  "markdown": {
    "breaks": false
  }
}
```

## 🔒 Security

* Improved plugin installation safety by enforcing the official registry during installs and replacing shell-based execution paths with safer process handling.
* Updated `@docmd/plugin-threads` to disable raw HTML inside thread comments whilst preserving standard markdown formatting (#136).

## 🛠 Improvements

* The build engine now exits with a non-zero status code when a plugin fails during lifecycle hooks, making CI failures easier to detect (#134).
* Added an optional `filePath` argument to the `onBeforeParse` and `onAfterParse` plugin hooks (#135).
* Reduced unnecessary console noise during hot configuration reloads.
* Updated the version switcher UI with improved dropdown labelling and icon alignment.
* Unified the workspace/project switcher UI (pills, dropdowns, and item list layout) with the version and language switchers for a consistent design language.

## 🐛 Fixes

* Fixed stale page caching issues when switching between projects in the development server.
* Fixed translation fetch errors on `noStyle` layouts.
* Fixed an issue where the copy-code icon could disappear on hover.

## Upgrade

```bash id="m0j1hv"
npm install -g @docmd/core
```
