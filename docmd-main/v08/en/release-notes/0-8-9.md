A focused reliability release. Two themes:

1. The plugin and template auto-installer is now resilient to packages that ship
   `import`-only `exports` fields. No more silent fallback failures for ESM-only
   packages.
2. The `@docmd/template-summer` and `docmd-search` packages now declare a
   `default` condition in their `exports`, matching the rest of the official
   plugin family. CI no longer pushes `docmd-search` to the `alpha` tag —
   it lands on `latest` like every other official package.

No public API changes. No breaking config changes. Purely a hardening release.

## 🐛 Bug fixes

### Plugin auto-install: resilient to `import`-only `exports`

The auto-installer in `@docmd/api` previously resolved installed packages with
Node's CommonJS `require.resolve`, then loaded the resolved path with
`await import(file://path)`. For packages that ship an `exports` field with
only an `import` condition, `require.resolve` throws
`ERR_PACKAGE_PATH_NOT_EXPORTED` and the build prints:

```
⬢ Could not load @docmd/template-summer after auto-install
```

even though the package is correctly installed. The retry path now uses
`await import(name)` directly, which honours the `exports` field natively and
works for any condition.

A defense-in-depth registry re-check is performed inside the retry path. The
set of names that can be auto-installed is unchanged — only the set of
`exports` conditions they can carry is larger.

### `@docmd/template-summer@0.8.9`: `default` exports condition added

`@docmd/template-summer` shipped with `exports["."]` declaring only the
`import` condition. The first-attempt `require.resolve` in the loader failed
for this reason, even when the package was correctly installed. The package
now declares:

```json
"exports": {
  ".": {
    "types": "./dist/index.d.ts",
    "import": "./dist/index.js",
    "default": "./dist/index.js"
  }
}
```

This matches the convention used by `@docmd/plugin-git`,
`@docmd/plugin-openapi`, and `@docmd/plugin-threads`. Update your lockfile
(`npm install`, `pnpm install`, `yarn install`, or `bun install`) after
upgrading `@docmd/core` to `0.8.9`.

### `docmd-search@0.1.0`: stable, not alpha, with `default` exports

`docmd-search` is no longer tagged `alpha`. The `0.1.0` release:

- Drops the `-alpha.1` suffix. The version on the `latest` tag is now
  `0.1.0`. The two `0.1.0-alpha.*` versions are deprecated — `npm install
  docmd-search` resolves to `0.1.0` as expected.
- Adds `"default": "./dist/index.js"` to both the `.` and `./client`
  subpaths in `exports`. The peer-dep `@huggingface/transformers` and the
  optional `onnxruntime-node` were already CJS-resolvable; the fix is
  specific to `docmd-search` itself.
- Removes the `--tag alpha` flag from the publish workflow. All future
  releases land on the default `latest` tag.

If you use semantic search in your docs (`plugins.search.semantic = true`),
update your `node_modules`:

```bash
rm -rf node_modules/docmd-search
npm install         # or pnpm/yarn/bun install
```

### Plugin loader: hardened `resolveDocmdSearch` fallback chain

The search plugin's resolver for the optional `docmd-search` peer dep now
uses an explicit `import` → `default` → `require` → `main` fallback chain
when reading the manifest, plus a manual `node_modules` walk as a backstop
if the `./package.json` subpath export is ever removed or pnpm's isolated
node_modules layout confuses the resolver. The function is now robust to
common install-layout variations without changing its return contract.

## 🔒 Security

No new attack surface. The trust boundary for auto-install remains the
official plugin registry allowlist plus the `@docmd/plugin-*` /
`@docmd/template-*` prefix check. The new `import(name)` retry path does
not extend the set of packages that can be installed — it only extends the
set of `exports` conditions a *registry-approved* package can carry.

The local-path plugin safety path (`safePath`, CWE-22/CWE-94) is unchanged.
The new defense-in-depth registry re-check inside the retry is an additional
guarantee, not a relaxation.

## 📚 Documentation updates

- **Plugins → Using Plugins** — the Auto-Installation section now notes the
  resilient resolver and the registry re-check.
- **Plugins → Search** — the Automatic Installation section mentions the
  hardened `resolveDocmdSearch` fallback chain.
- **Development → Building Plugins** and **Development → Building Templates**
  — new "ESM exports — the `default` condition" subsection tells authors
  they must include `"default": "./dist/index.js"` in their `exports` for
  the auto-install path to behave cleanly.
