### ✨ Highlights

This release introduces semantic search as an alpha preview, automatic `robots.txt` generation in the SEO plugin, fixes Mermaid C4Context diagrams, and adds a config upgrade command for seamless migration.

### Semantic Search (Alpha Preview)

docmd now supports semantic search powered by local embeddings, enabling context-aware search results beyond simple keyword matching.

**Key Features:**

*   **Context Understanding** — Queries like "authentication" find relevant pages using "login" or "sign-in" terminology
*   **Typo Tolerance** — Natural typo handling without fuzzy matching logic
*   **Concept Discovery** — Searching "API" returns relevant endpoint documentation
*   **100% Private** — All embeddings run client-side; no external API calls
*   **Offline Ready** — Works entirely without network connectivity

Enable semantic search by installing `docmd-search` and adding `semantic: true` to your configuration:

```bash
npm install docmd-search
```

```json
{
  "plugins": {
    "search": {
      "semantic": true
    }
  }
}
```

The plugin automatically falls back to keyword search if `docmd-search` is not installed, ensuring your documentation remains searchable.

**Note:** This is an alpha preview. English-only models are currently available, with multilingual support planned for future releases.

### SEO Plugin: robots.txt Auto-Generation

The SEO plugin now automatically generates a `robots.txt` file during the build process if one doesn't exist.

**Features:**

*   **Smart Defaults** — Includes `User-agent: *` and `Allow: /` by default
*   **Sitemap Reference** — Automatically adds sitemap URL when `config.url` is set
*   **AI Bot Control** — By default, AI bots are allowed to index content (`aiBots: true`). Set `plugins.seo.aiBots: false` to block AI training bots
*   **Non-Destructive** — Existing `robots.txt` files are never overwritten

**Default Behaviour (AI bots allowed):**

```json
{
  "plugins": {
    "seo": {
      "aiBots": true
    }
  }
}
```

This is the default — no configuration needed. AI bots like GPTBot and ChatGPT-User can index your documentation.

**Block AI Training Bots:**

```json
{
  "plugins": {
    "seo": {
      "aiBots": false
    }
  }
}
```

This generates `User-agent` directives for GPTBot, ChatGPT-User, Google-Extended, CCBot, and other AI crawlers to prevent indexing.

### Mermaid C4Context Fix

C4Context diagrams now render correctly instead of displaying as blank white boxes.

**Root Cause:** The `mermaid.render()` API omits the `xmlns:xlink` namespace declaration when generating SVGs with `xlink:href` attributes (used by C4Context person icons). DOMParser's strict XML parser fails on undeclared namespaces, causing rendering failures.

**Solution:** A new `fixSvgNamespaces()` utility injects the missing namespace declaration before DOMParser processes the SVG, ensuring all Mermaid diagram types render correctly.

Thanks to [@sinsombat](https://github.com/sinsombat) for the comprehensive fix and test suite (10/10 tests passing).

### Config Upgrade Command

A new `--upgrade` flag has been added to the `docmd migrate` command. Running it automatically rewrites your configuration file to the modern schema in place.

```bash
npx @docmd/core migrate --upgrade
```

The upgrade handles the following legacy key mappings automatically:

| Legacy Key | Modern Key |
| :--- | :--- |
| `projects` (top-level) | `workspace.projects` |
| `siteTitle` | `title` |
| `siteUrl` / `baseUrl` | `url` |
| `srcDir` / `source` | `src` |
| `outputDir` | `out` |
| `defaultLocale` | `i18n.default` |

The original file is rewritten in place. A backup is not required as the migration is non-destructive — it only renames keys and does not alter their values.

### TOC HTML Entity Decoding

Heading text containing special characters (such as `<`, `>`, `&`, or smart quotes) is now decoded correctly in the Table of Contents sidebar. Previously, headings with these characters displayed raw HTML entities (e.g., `&amp;`) instead of the intended characters.

### Changelog

**New Features:**

1.  **Semantic Search**: Added alpha preview of semantic search with local embeddings and automatic fallback to keyword search.
2.  **SEO Plugin**: Implemented automatic `robots.txt` generation with AI bot blocking support.
3.  **Search: `showFilters` option** — Set `showFilters: false` in the search plugin config to hide the version filter bar above search results. Useful for single-version sites or when a cleaner search UI is preferred.
4.  **Search: `showConfidence` option** — Set `showConfidence: true` to display a `%` match badge on each semantic search result. Green for ≥ 85%, accent-blue otherwise. Styled entirely via CSS classes (`.search-result-confidence`, `.confidence-high`).
5.  **Search: right-aligned result meta** — Version pill and confidence badge are now grouped in a `.search-result-meta` flex container, always right-aligned regardless of whether versioning is enabled.

**Bug Fixes:**

1.  **Mermaid**: Fixed C4Context diagrams rendering as blank white boxes by adding missing `xmlns:xlink` namespace declarations.
2.  **Live Editor**: Fixed template render crash caused by `workspace` variable being accessed before initialisation.
3.  **TOC Rendering**: Fixed HTML entity decoding in Table of Contents so special characters display correctly.
4.  **Dev Server**: Fixed spurious page reloads caused by `fs.watch` firing multiple events per save on macOS. Rebuild now uses a 600ms debounce — the browser only reloads after 600ms of inactivity, not while the user is still typing/saving.

**Improvements:**

1.  **Migration CLI**: Added `docmd migrate --upgrade` for automated config modernisation.
2.  **UI Polish**: Restyled Project Switcher to align with i18n and appearance buttons.
3.  **Dev Server**: Config and `navigation.json` changes now trigger a fast targeted rebuild instead of a full restart; the dev URL is automatically opened in the default browser on server start.
