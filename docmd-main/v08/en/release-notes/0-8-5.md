### ✨ Highlights

This release resolves a critical rendering error in the Live Editor service, introduces a `migrate --upgrade` command for automated config modernisation, and polishes the Project Switcher UI.

### Live Editor fix

A template rendering error affecting the Live Editor service (`live.docmd.io`) has been resolved.

The `workspace` variable was accessed inside `layout.ejs` before it was defined in the browser rendering context, causing the following error on every page render:

```
Error: Template Render Error: Cannot access 'workspace' before initialization.
```

The browser entry point now correctly passes `workspace: null` to the template renderer, resolving the crash for all users of the live preview environment.

### Config upgrade command

A new `--upgrade` flag has been added to the `docmd migrate` command. Running it automatically rewrites your configuration file to the modern schema in place.

```bash
npx @docmd/core migrate --upgrade
```

The upgrade handles the following legacy key mappings automatically:

| Legacy key | Modern key |
| :--- | :--- |
| `projects` (top-level) | `workspace.projects` |
| `siteTitle` | `title` |
| `siteUrl` / `baseUrl` | `url` |
| `srcDir` / `source` | `src` |
| `outputDir` | `out` |
| `defaultLocale` | `i18n.default` |

The original file is rewritten in place. A backup is not required as the migration is non-destructive - it only renames keys and does not alter their values.

### TOC HTML entity decoding

Heading text containing special characters (such as `<`, `>`, `&`, or smart quotes) is now decoded correctly in the Table of Contents sidebar. Previously, headings with these characters displayed raw HTML entities (e.g., `&amp;`) instead of the intended characters.

### Changelog

1. **Live Editor**: Fixed a template render crash caused by the `workspace` variable being accessed before initialisation in the browser rendering context.
2. **Config upgrade**: Added `docmd migrate --upgrade` to automatically rewrite legacy configuration keys to the modern schema.
3. **TOC rendering**: Fixed HTML entity decoding in Table of Contents heading text so special characters display correctly.
4. **UI polish**: Restyled the options-menu Project Switcher to align visually with the i18n and appearance buttons, and corrected button ordering.
