## 问题

搜索引擎根据内容结构来排序。如果一份高质量的指南使用了诸如"简介"或"步骤 1"这类通用标题，搜索引擎可能无法为其中的核心关键词赋予足够权重。相关页面会被埋在搜索结果里，让期望即时得到答案的用户感到受挫。

## 为什么重要

用户搜索的通常是具体的技术词（例如 "authentication token" 或 "deployment limit"），而非完整句子。如果您的文档结构没有突出这些词，搜索引擎就无法自信地为您的内容排序。较高的搜索相关性能够减少大量支持工单。

## 方法

组织好 Markdown，使搜索索引器能自动识别并优先处理核心概念。docmd 的搜索引擎会为页面的 `title`、`description` 与 `headers` 赋予高于正文的权重。优化这些结构化元素能显著改善可发现性。

## 实现

### 1. 优化 Frontmatter 元数据

使用 [Frontmatter](../../content/frontmatter.md) 块提供显式的关键词和描述性摘要。[Search 插件](../../plugins/search.md) 会对这些字段建立索引，从而在搜索 UI 中提供更好的结果与更有用的摘要。

```yaml
---
title: "AWS S3 存储配置"
description: "如何为 AWS S3 集成配置 IAM 角色与存储桶权限。"
keywords: ["aws", "s3", "storage", "iam", "cloud"]
---
```

### 2. 使用语义化标题

避免使用通用标题名。在标题中加入相关关键词，为用户和搜索引擎提供上下文。

*   **低相关性**：`## 步骤 1：配置`
*   **高相关性**：`## 步骤 1：配置 AWS IAM 角色`

### 3. 使用标注突出关键信息

将 [标注容器 (Callout)](../../content/containers/callouts.md) 用于关键警告或"小贴士"可提升搜索相关性。标注内的内容会被语义隔离，并由索引器赋予不同权重，以突出重要的排错步骤。

## 取舍

针对搜索相关性进行优化要求写作时保持纪律。随着产品的演进，Frontmatter 中的关键词若不定期审查便会过时。此外，在标题中堆砌过多关键词（keyword stuffing）会让文档显得重复且不自然。请在 SEO 与可读性之间寻求平衡。
