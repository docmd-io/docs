### ✨ 亮点

本版本带来了**自定义模板插件 API** —— 一种以独立插件形式发布替代布局的完全模块化方式 —— 由 **Summer** 领衔登场，它是首个官方社区模板（可通过新的 `docmd add summer` 命令安装）。两个**核心 UI 功能**（Cookie 同意对话框和全站公告横幅）现在已内置于 `@docmd/ui`，每个模板都能在不依赖单独插件的情况下采用它们。还包含新的资源优先级系统、**Docker `:latest` 标签修复**（开箱即用），以及零破坏性变更。

## 🧩 自定义模板插件 API

模板现在就是插件。一个模板包声明 `capabilities: ['template']` 并附带 `templates[]` 数组（包含 `.ejs` 文件覆盖）。`@docmd/ui` 中的解析器（新引入的 `resolveTemplate()`）遵循以下解析链：

1. `frontmatter.template`（按页）
2. `config.templates[glob]`（按分区，未来）
3. `config.theme.template`（全站）
4. `@docmd/ui` 内置默认

任何一步失败或解析到的文件在磁盘上不存在时，解析器会回退到默认，并输出一次 TUI 警告。模板通过 `templateAssets[]` 提供自己的 CSS/JS，它们以优先级 `10` 加载，因此用户的 `customCss`（优先级 `15`）始终胜出。模板**不得**在 CSS 中使用 `!important`。

**解析示例**

```js
// template-summer/index.js
export default {
  plugin: {
    name: 'template-summer',
    version: '0.1.0',
    capabilities: ['template'],
  },
  templates: [
    { type: 'layout', templatePath: '...' },
    { type: 'sidebar', templatePath: '...' },
  ],
  templateAssets: [
    { type: 'css', path: '.../summer.css', priority: 10, position: 'head' },
  ],
};
```

```json
// docmd.config.json
{ "theme": { "template": "summer" } }
```

模板也可以省略 14 个受支持插槽中的任意一个（`layout`、`404`、`toc`、`navigation`、`sidebar`、`header`、`footer`、`menubar`、`options-menu`、`project-switcher`、`version-dropdown`、`language-switcher`、`banner`、`cookie-consent`）；未提供的插槽将由解析器自动回退到默认。

**通过 frontmatter 按页覆盖**

```markdown
---
title: "更新日志"
template: "template-changelog"
---
```

首个官方模板包 —— `@docmd/template-summer`（**Summer** 布局）—— 与本版本同步发布，可通过标准的 `docmd add` 流水线安装。

## ☀️ Summer 模板预览

首个基于自定义模板 API 构建的官方模板是 **Summer** —— 一个明亮、轻盈、充满夏日气息的布局，自带顶部搜索栏、柔和光晕和更紧凑的卡片网格。通过标准的 `docmd add` 流水线安装：

```bash
docmd add summer
```

这将从 npm 安装 `@docmd/template-summer` 并自动更新您的 `docmd.config.json`：

```diff
  module.exports = defineConfig({
+   theme: {
+     template: "summer"
+   },
    // ... 您现有的配置
  });
```

与普通插件不同，Summer 写入的是 `theme.template`（而不是 `plugins.*`），并且会**覆盖**之前设置的模板 —— 不存在叠加。运行 `docmd remove summer` 可恢复默认。Summer 自带一份专用的 `404.ejs`；当该插槽被省略时，默认的 404 用作兜底。

> **发布顺序：** `@docmd/template-summer@0.8.6` 会在 monorepo 其余部分之前发布，以满足 npm 的 OIDC 首次发布要求（必须从维护者的本地机器发起）。与 monorepo 在 `0.8.7` 同步发布的版本功能上完全一致。

## 🍪 Cookie 同意对话框

一个轻量、无障碍的 Cookie 同意对话框已随默认 UI 一起提供。在您的配置中启用即可，无需安装插件。

```json
{
  "cookie": {
    "enabled": true,
    "message": "We use cookies to ensure you get the best experience.",
    "policyUrl": "/privacy",
    "position": "bottom-right",
    "dismissible": true,
    "expiryDays": 180
  }
}
```

| 字段 | 默认值 | 说明 |
|---|---|---|
| `enabled` | `true`（当 `cookie` 对象存在时） | 总开关。 |
| `message` | 翻译键 `cookieMessage` | 对话框正文。 |
| `acceptText` | 翻译键 `cookieAccept` | 接受按钮文本。 |
| `declineText` | 翻译键 `cookieDecline` | 拒绝按钮文本。 |
| `policyUrl` | `null` | 可选，链接到您的隐私政策。 |
| `position` | `bottom` | `bottom` \| `bottom-left` \| `bottom-right` \| `center` |
| `dismissible` | `true` | 显示关闭 (X) 按钮。 |
| `expiryDays` | `180` | 选择在 `localStorage` 中的保留时长。 |

如果未设置 `config.cookie`，**不会渲染任何对话框** —— 这是有意为之的按需启用。用户作出选择后，`window` 上会触发 `docmd:cookie-consent` `CustomEvent`，便于模板和插件响应。

## 📢 全站公告横幅

一个可关闭的横幅，位于 menubar 之上、页头之下。支持内联 Markdown（`**加粗**` 和 `` `代码` ``），也可传入原始 `html`。

```json
{
  "layout": {
    "banner": {
      "content": "**v0.9 周五发布** —— 阅读公告。",
      "type": "info",
      "dismissible": true,
      "link": { "text": "了解更多", "url": "/blog/v0-9" },
      "icon": "megaphone"
    }
  }
}
```

| 字段 | 默认值 | 说明 |
|---|---|---|
| `content` | `""` | 内联 Markdown 文本。与 `html` 互斥。 |
| `html` | `""` | 原始 HTML。优先级高于 `content`。 |
| `type` | `"info"` | `info` \| `success` \| `warning` \| `danger` |
| `dismissible` | `true` | 显示关闭 (X) 按钮。 |
| `link` | `null` | `{ text, url }`，CTA 链接。 |
| `icon` | `null` | 左侧显示的 Lucide 图标名。 |

"已关闭" 状态保存在 `sessionStorage` 中，因此在新的访问中横幅会再次出现。如果未设置 `config.layout.banner`，**不会渲染任何横幅** —— 这是有意为之的按需启用。

## 更新日志

### 新功能

- **自定义模板插件 API** —— 模板现在是带有 `capabilities: ['template']`、`templates[]` 和 `templateAssets[]` 钩子的插件；新的 `resolveTemplate()` 解析器带 4 步兜底链（按页 → 按分区 → 全站 → 默认）。
- **`@docmd/api` 和 `@docmd/ui` 中的新类型**：`Capability: 'template'`、`TemplateHook`、`TemplateAssetHook`、`TemplateSlot`、`TemplateResolutionContext`、`ResolvedTemplate`、`Asset`、`AssetKind`、`AssetPosition`。
- **`PluginHooks` 上的新注册表字段**：`templates` 和 `templateAssets`；新的 `PluginModule.templates?` / `templateAssets?` 导出。
- **`@docmd/ui` 中的新解析器 API**：`resolveTemplate()` 和 `clearTemplateResolverCache()`。
- **`docmd add summer`** —— 一等模板安装命令，写入 `theme.template`（而非 `plugins.*`）；`docmd remove summer` 清除它。
- **Cookie 同意对话框** 内置到 `@docmd/ui` —— 通过 `config.cookie` 按需启用；触发 `docmd:cookie-consent` `CustomEvent`。
- **全站公告横幅** 内置到 `@docmd/ui` —— 通过 `config.layout.banner` 按需启用；关闭状态保存在 `sessionStorage`。
- **资源优先级系统** —— 确定的 CSS/JS 加载顺序：基础(0) → 主题(5) → 模板(10) → customCss(15) → 插件(20)。
- **14 个受支持的模板插槽**：`layout`、`404`、`toc`、`navigation`、`sidebar`、`header`、`footer`、`menubar`、`options-menu`、`project-switcher`、`version-dropdown`、`language-switcher`、`banner`、`cookie-consent`。

### 缺陷修复

- **Docker `:latest` 标签缺失** —— 已发布的镜像缺少 `latest` 标签，因为 workflow 的 `enable` 表达式要求 `main` 分支上的 `push` 事件，但该 workflow 没有 `push` 触发器。已将所有文档固定到 `:0.8.6`，并重写 [`.github/workflows/docker-publish.yml`](https://github.com/docmd-io/docmd/blob/dev-0.8.7/.github/workflows/docker-publish.yml) 以 (a) 在 `release` 事件上自动打 `:latest` 标签，(b) 重新加入 `push: branches: [main]` 触发器以重建 `:edge`，(c) 暴露一个手动 `tag_latest` 输入用于补打标签。完整背景请参阅 [bug 报告审计](https://github.com/docmd-io/docmd/blob/dev-0.8.7/bug-report.md)。
- **partial 中的 i18n 字符串未翻译** —— `menubar`、`options-menu`、`version-dropdown`、`language-switcher`、`project-switcher` 和 `toc` 这些 partial 调用了 `t('…')`，但布局从未将 `t` 函数传递给它们（只有 `banner`、`footer` 和 `cookie-consent` 接收到了）。在 `layout.ejs` 的每个 `include()` 调用中加入 `t: typeof t === 'function' ? t : null`，使所有 partial 都能正确翻译。同时为 7 个语言文件（`en`/`de`/`es`/`fr`/`hi`/`ja`/`zh`）新增了 13 个缺失的翻译键（`close`、`cookieAccept`、`cookieConsent`、`cookieDecline`、`cookieMessage`、`cookiePolicy`、`goBack`、`lastUpdated`、`pageNotFoundMsg`、`readMore`、`recentCommits`、`searchPlaceholder`、`selectProject`）。
- **代码块复制按钮悬浮位置错误** —— 按钮原本锚定在内部 `.code-wrapper`（通过 `docmd-main.js` 中的内联 `style.position = 'relative'` 设置），仅定位到 `<pre>` 的右上角，忽略了上方标题栏。现在按钮锚定到外部 `.docmd-code-block-wrapper`（其已在 CSS 中设为 `position: relative`），因此能相对于整个代码块（含其标题）正确定位。同时，对于解析器已经渲染了内部 wrapper 的代码块，跳过创建多余的内部 wrapper。
- **Summer 模板：短内容下方出现巨大空白** —— `.summer-content` 设置了 `flex: 1 0 auto`，会使文章内容拉伸填满网格单元，导致短页面在文章和页脚之间留下巨大的空白间隙。已移除 grow，使内容保持自然高度；页脚仍通过现有的 `margin-top: auto` 紧贴底部。
- **Summer 模板：代码块标题栏缺少复制按钮** —— `summer.js` 中的 `summerCodeblocks()` 只在 `.code-wrapper` 内查找 `.copy-code-button`，但 `docmd-main.js` 中的 `injectCopyButtons()` 会将按钮直接追加到 `.docmd-code-block-wrapper`（没有内部 `.code-wrapper`），针对带解析器渲染标题的代码块（如 ```` ```js "main.ts" ````）。按钮仍内联显示在 `<pre>` 下方，而不会进入标题栏。`summerCodeblocks()` 现在在 wrapper 自身中查找按钮，因此两种代码块路径下的标题栏都能获得复制控件。
- **Summer 模板：changelog 时间线与条目圆点错位** —— `.changelog-timeline` 的两条 CSS 规则（一条具体、一条通用）部分重叠：具体规则将线条设到 `left: 0`，但通用规则给时间线容器加了 `padding-left: 32px`，把每个条目向右推 32px，圆点因此漂浮在线条之外 28px。具体规则的线条偏移现已改为 `left: 32px`，线条中心与每个条目的圆点中心对齐。
- **Docker 镜像：快速启动报 "No documentation directory or configuration found"** —— 镜像的 `WORKDIR` 是 `/docs`，默认 `CMD` 为 `docmd dev --host 0.0.0.0`，但镜像中的 `/docs` 是空的（种子模板在 `/template` 中）。已新增一个 `docker-entrypoint.sh`：仅当 `/docs` 完全为空时才将 `/template` 拷贝到 `/docs`（这样不会覆盖用户挂载的卷），然后 `exec` 用户传入的命令。镜像现在开箱即用，无需挂载卷。
- **Docker 镜像：健康检查始终报告异常** —— 镜像的 `HEALTHCHECK` 硬编码 `curl http://localhost:3000/`，但 `docmd dev` 在 3000 被占用时会自动递增到 3001+（例如主机上已有其他开发服务器在运行）。已将内联 curl 替换为 `docker/docker-healthcheck.sh`，探测 3000–3005，一旦任一端口有响应即返回 0。
- **条件性插件资源加载破坏 SPA 导航** —— 当页面有条件地发出插件的 `<script src="...">` 标签时（例如 Mermaid 的 `init-mermaid.js` 仅在带 `class="mermaid"` 的页面发出），脚本标签被渲染到 `<main>` 之后的 `<body>` 中，因此 SPA 导航的内容切换（仅替换 `.main-content` 和固定选择器列表）会让新标签停留在已解析的 `doc` 中，永远不会进入实时文档。从非 mermaid 页面导航到 mermaid 页面会导致 `.mermaid` 区块未被渲染（无监听器、无 IIFE、无初始化）。直接加载和刷新可以工作，因为脚本已在初始 HTML 中；KaTeX（仅 CSS，在 `<head>` 中）可以工作，因为现有的 head 资源同步处理了它。已在 `navigateTo()` 中加入 body 脚本同步，与 head 同步对应：对于新 HTML 中每个已存在于实时文档之外的 `body > script[src]`，构造一个新的 `<script>` 元素（不是 `cloneNode` —— 浏览器会跳过克隆的脚本），使用解析后的绝对 URL 并追加它。插件自身的 IIFE 处理当前页面的渲染（当 `document.readyState !== "loading"` 时直接调用其 setup）；它注册的 `docmd:page-mounted` 监听器覆盖后续导航。该同步完全是通用的 —— 没有插件名、没有关键字、没有硬编码路径 —— 因此对未来任何插件资源都适用。
- **Summer 模板：带标题的代码块丢失内边距和字体** —— `.code-wrapper>pre, .docmd-code-block-wrapper>.code-wrapper>pre` 选择器只匹配在外层 wrapper 和 `<pre>` 之间存在内部 `.code-wrapper` 的代码块。带标题的代码块（` ```js "title" ` `）的 `<pre>` 是 `.docmd-code-block-wrapper` 的直接子元素 —— 没有内部 `.code-wrapper`，因为 `injectCopyButtons` 将复制按钮直接追加到 wrapper。该选择器遗漏了这些 `<pre>`，因此它们继承了零内边距和浏览器默认的等宽字体，而不是 Summer 的 token（代码紧贴 wrapper 边缘）。已将 `.docmd-code-block-wrapper>pre` 加入 pre 与 pre-code 选择器，使所有三种代码块形态（`.code-wrapper > pre`、`.docmd-code-block-wrapper > .code-wrapper > pre`、`.docmd-code-block-wrapper > pre`）都能获得 Summer 的内边距 (16px 18px)、JetBrains Mono 字体 (0.84rem)、行高 (1.65) 和前景色。
- **Summer 模板：代码块规则残留 `!important`，违反模板指南** —— 文件自身的头部声明"不要使用 `!important`。用户的 customCss（优先级 15）必须始终胜出，以便用户无需 fork 模板就能重新设置任何选择器"，但 pre/code-wrapper 规则在 8 个属性上使用了 `!important`。这个 `!important` 是 `docmd-main.css` 和 `summer.css` 同时加载时的遗留产物；在当前设计中，Summer 布局**不会**加载 `docmd-main.css`（依布局头部注释），而 Summer 的选择器比基础 `pre` 规则具有更高的特异性，因此 `!important` 已不再必要。已从代码块 pre、code-wrapper 和 pre-code 规则中移除。经实测：Summer 的样式仍然生效（内边距、字体、透明背景），用户的 `customCss` 现在可以正确覆盖 Summer 而无需使用 `!important`（如指南所期望）—— 而带 `!important` 的 `customCss` 仍然胜出（如它本应如此）。
- **侧边栏导航展开/收起在真实链接上静默失效** —— `navigation.ejs` 把 `.nav-group` 同时放在虚拟的 `<span class="nav-label">` 和真实的 `<a class="nav-group">`（带 `href` 的）上。全局点击处理函数只匹配 `.nav-label, .collapse-icon-wrapper`，因此点击真实链接的标签文本（如"配置"）会静默无效 —— 只有箭头图标能触发。已更新切换处理函数以同时匹配 `.nav-group`，因此点击标签文本也能展开/收起分组。为避免切换处理函数和 SPA 路由同时触发（这会直接导航离开并在新页面重置侧栏状态），SPA 路由现在跳过 `li.collapsible` 内的 `<a class="nav-group">`。已在 playground（Summer 模板）和文档站（默认主题）上验证：收起 → 点击 `<a>` → 分组展开，无 SPA 导航；展开 → 点击 `<a>` → 分组收起（aria-expanded=false，子菜单动画关闭）；叶子链接（无 `.nav-group`）仍按以前的方式 SPA 导航。
- 资源管线现在使用稳定的优先级排序，消除了多个插件同时发出样式时 CSS 加载顺序的非确定性。
- `loadPlugins()` 现在在每次构建之间重置 `templates` 和 `templateAssets` 数组，与其他所有钩子数组的生命周期一致。
- `template-resolver.ts` 对插件名宽松处理 `@docmd/` 与 `template-` 前缀（`summer`、`template-summer`、`@docmd/template-summer` 三者解析结果一致）。
- 新的 `Asset` 接口对之前 `getAssets()` 返回的 `any[]` 进行了类型检查。

  | `theme.codeHighlight` | `true` | `<pre>` 块启用语法高亮 |
- **粘性底部布局** —— `.page-navigation`（上一页/下一页）和 `.page-footer-actions` 现在通过 flex `margin-top: auto` 固定在内容区底部，因此在短页面上不再漂浮在中间。固定在文章底部、页脚之上。
- **更纤细的页头与 menubar** —— 默认页头高度与 menubar 高度都从 52–54px 缩减到 44px。页头内边距从 `var(--space-3)` 调整为 `var(--space-2) var(--space-3)`。减少纵向留白，更多内容空间。
- **更小的永久链接图标** —— `heading-anchor` SVG 现在为 `0.85em`（原为 `1.1em`）。默认更克制；仍会在标题悬停时出现。
- **更精致的默认 404 页面** —— 内置的 `404.ejs` 现在采用玻璃卡片布局与柔和光晕背景（绑定到 `--link-color` / `--glow-color`），带渐变风格的巨大 "404" 标题，以及 **Return home** 与 **Go back** 两个操作按钮（每个都带 Lucide 图标）。使用默认主题 token 复刻了 Summer 模板的 404 设计语言，因此可与任何 `theme.name` 搭配使用。`404` 字形本身通过 `errorCode404` 翻译键渲染，以保证屏幕阅读器一致性。
- **404 页面完整翻译** —— 默认 404 页面上的所有用户可见字符串（`errorCode404`、`pageNotFound`、`pageNotFoundMsg`、`returnHome`、`goBack`）现已通过 `t()` 包裹并具备英文回退，`<html lang="…">` + `dir="…"` 属性从 `locals.activeLocale` 派生，以便在 RTL 语言（例如日后加入的阿拉伯语、希伯来语）下正确渲染。相同的键已存在于所有 7 个语言版本中。
- **卡片标题区域缩小** —— `.card .card-title` 块现在使用 `0.95rem` 字号（原为 `1.125rem`），`var(--space-2) var(--space-4)` 内边距（原为 `var(--space-3) var(--space-6)`），底部外边距 `var(--space-3)`（原为 `var(--space-4)`）。`grids` 容器中堆叠的功能网格更紧凑，卡片不再显得头重脚轻。
- **三个新翻译键** —— `poweredBy`（"Powered by" / 本地化）、`viewSource`（"View source" / 本地化）、`goToTop`（"Go to top" / 本地化）已加入全部 7 个语言文件。可在模板中使用，或通过 `docmd.config.json` 中的 `i18n.locales[].translations` 进行覆盖。
- **通用 UI 现代化** —— 新增 4 个光晕 token（`--glow-color`、`--glow-color-strong`、`--halo-soft`、`--halo-soft-strong`），并一致地应用于卡片、按钮、聚焦环、激活态侧栏胶囊、步骤编号、404 页面和 hero。卡片悬停现在结合更柔和的阴影 + 光晕 + `color-mix()` 边框色。常见交互元素统一添加 `0.15s ease` 过渡。`:focus-visible` 现在获得光晕聚焦，键盘聚焦更明显。
- **文本与颜色对比度提升** —— 浅色 `--text-color` `#3f3f46` → `#27272a`；深色 `--text-color` `#cacaca` → `#d4d4d8`；深色 `--text-muted` `#71717a` → `#94949e`；深色 `--bg-color` `#111` → `#0d0d0f`；深色 `--link-color` `#068ad5` → `#38bdf8`。所有值对其各自背景均通过 WCAG AA。
- **激活态侧边栏链接重新设计** —— 用链接左缘 3px "节点在线" 胶囊（垂直居中、与现有 `.submenu` 丝线边框部分重叠）取代了 `inset 2px 0` 左边框。视觉上是一个清晰的激活标记，又不会与侧边栏的分隔线争抢注意力。
- **步骤永久链接** —— 每个步骤 `<li>` 现在都有一个全局唯一的 `id="step-N"` 以及一个隐藏的 `.step-permalink` 锚点（Lucide `link-2` 图标），在悬停/聚焦时淡入。`scroll-margin-top` 使锚点跳转落在粘性页头之下，避免步骤标题被遮挡。
- `theme.name`（CSS 主题）继续可用且能正确与模板叠加。
- `customCss` / `customJs` 数组仍然最后加载。
- `frontmatter.template` 之前在旧的 `templates` 查找表中解析 —— 该查找表对仍然存在的键（`docmd-search`、`no-style`）保持保留；新键现在走解析器。
- `frontmatter.noStyle` 继续使用默认的 `no-style` 模板。
- 0.8.7 之前注册的所有钩子仍按相同顺序触发。
- 现有 `Asset` 别名（`src` / `dest` / `location` / `attributes`）保留为已弃用，以兼容旧插件。
