### ✨ 亮点

此发布解决了实时编辑器服务中的关键渲染错误，引入了用于自动化配置现代化的 `migrate --upgrade` 命令，并对项目切换器 UI 进行了优化。

### 实时编辑器修复

实时编辑器服务（`live.docmd.io`）中的模板渲染错误已被解决。

`workspace` 变量在浏览器渲染上下文中的定义之前在 `layout.ejs` 内被访问，导致每个页面渲染时出现以下错误：

```
Error: Template Render Error: Cannot access 'workspace' before initialization.
```

浏览器入口点现在正确地将 `workspace: null` 传递给模板渲染器，解决了所有实时预览环境用户的崩溃问题。

### 配置升级命令

已向 `npx @docmd/core migrate` 命令添加了新的 `--upgrade` 标志。运行它会自动将你的配置文件重写为现代架构。

```bash
npx @docmd/core migrate --upgrade
```

升级自动处理以下遗留键映射：

| 遗留键 | 现代键 |
| :--- | :--- |
| `projects`（顶级） | `workspace.projects` |
| `siteTitle` | `title` |
| `siteUrl` / `baseUrl` | `url` |
| `srcDir` / `source` | `src` |
| `outputDir` | `out` |
| `defaultLocale` | `i18n.default` |

原始文件就地重写。不需要备份，因为迁移是非破坏性的 - 它只重命名键，不改变其值。

### TOC HTML 实体解码

包含特殊字符（如 `<`、`>`、`&` 或聪慧引号）的标题文本现在在目录侧边栏中正确解码。以前，具有这些字符的标题显示原始 HTML 实体（例如 `&amp;`）而不是预期的字符。

### 更新日志

1. **实时编辑器** - 修复了模板渲染崩溃，由于 `workspace` 变量在浏览器渲染上下文中初始化之前被访问。
2. **配置升级** - 添加了 `npx @docmd/core migrate --upgrade` 以自动将遗留配置键重写为现代架构。
3. **TOC 渲染** - 修复了目录标题文本中的 HTML 实体解码，使特殊字符正确显示。
4. **UI 优化** - 重新设计了选项菜单项目切换器，以与 i18n 和外观按钮在视觉上对齐，并更正了按钮顺序。
