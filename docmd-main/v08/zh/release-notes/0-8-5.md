### ✨ 亮点

本次发布引入了语义搜索 Alpha 预览功能、SEO 插件自动 `robots.txt` 生成、修复 Mermaid C4Context 图表渲染问题，并新增了用于无缝迁移的配置升级命令。

### 语义搜索（Alpha 预览）

docmd 现在支持基于本地向量嵌入的语义搜索，能够提供超越简单关键词匹配的上下文感知搜索结果。

**核心特性：**

*   **上下文理解** — 搜索"身份验证"可找到使用"登录"或"注册"等不同表述的相关页面
*   **拼写容错** — 无需模糊匹配逻辑，自然处理拼写错误
*   **概念关联发现** — 搜索"API"可返回相关的接口文档
*   **100% 私有** — 所有嵌入运算均在客户端本地执行，不调用外部 API
*   **离线可用** — 完全无需网络连接即可工作

通过安装 `docmd-search` 并在配置中添加 `semantic: true` 来启用语义搜索：

```bash
npm install docmd-search
```

```json
{
  "plugins": {
    "search": {
      "semantic": true
    }
  }
}
```

如果未安装 `docmd-search`，插件会自动降级到关键词搜索，确保文档始终可被搜索。

**注意：** 此功能为 Alpha 预览版。目前仅提供英语模型，多语言支持计划在未来版本中推出。

### SEO 插件：robots.txt 自动生成

SEO 插件现在会在构建过程中自动生成 `robots.txt` 文件（前提是该文件不存在）。

**功能特性：**

*   **智能默认值** — 默认包含 `User-agent: *` 和 `Allow: /`
*   **站点地图引用** — 设置 `config.url` 后自动添加站点地图 URL
*   **AI 爬虫管理** — 默认允许 AI 爬虫索引内容（`aiBots: true`）。设置 `plugins.seo.aiBots: false` 可阻止 AI 训练爬虫
*   **非破坏性** — 已存在的 `robots.txt` 文件不会被覆盖

**默认行为（允许 AI 爬虫）：**

```json
{
  "plugins": {
    "seo": {
      "aiBots": true
    }
  }
}
```

这是默认配置，无需任何额外设置。GPTBot 和 ChatGPT-User 等 AI 爬虫可以索引你的文档。

**阻止 AI 训练爬虫：**

```json
{
  "plugins": {
    "seo": {
      "aiBots": false
    }
  }
}
```

这将为 GPTBot、ChatGPT-User、Google-Extended、CCBot 等 AI 爬虫生成 `User-agent` 禁止指令。

### Mermaid C4Context 修复

C4Context 图表现在能够正确渲染，不再显示为空白白框。

**根本原因：** `mermaid.render()` API 在生成包含 `xlink:href` 属性的 SVG（C4Context 人物图标使用此属性）时，会省略 `xmlns:xlink` 命名空间声明。DOMParser 的严格 XML 解析器在遇到未声明的命名空间时会失败，导致渲染错误。

**解决方案：** 新增的 `fixSvgNamespaces()` 工具函数会在 DOMParser 处理 SVG 之前注入缺失的命名空间声明，确保所有 Mermaid 图表类型都能正确渲染。

感谢 [@sinsombat](https://github.com/sinsombat) 提供的完整修复方案和测试套件（10/10 测试全部通过）。

### 配置升级命令

`docmd migrate` 命令新增了 `--upgrade` 标志。运行该命令可自动将配置文件就地重写为现代格式。

```bash
npx @docmd/core migrate --upgrade
```

升级会自动处理以下遗留键映射：

| 遗留键 | 现代键 |
| :--- | :--- |
| `projects`（顶级） | `workspace.projects` |
| `siteTitle` | `title` |
| `siteUrl` / `baseUrl` | `url` |
| `srcDir` / `source` | `src` |
| `outputDir` | `out` |
| `defaultLocale` | `i18n.default` |

原始文件会就地重写。由于迁移是非破坏性的（只重命名键，不修改值），因此不需要备份。

### TOC HTML 实体解码

包含特殊字符（如 `<`、`>`、`&` 或弯引号）的标题文本，现在能在目录侧边栏中正确解码显示。此前，包含这些字符的标题会显示原始 HTML 实体（如 `&amp;`）而非正确字符。

### 更新日志

**新功能：**

1.  **语义搜索**：新增基于本地向量嵌入的语义搜索 Alpha 预览，并支持自动降级到关键词搜索。

3.  **TOC 渲染**：修复了目录中 HTML 实体解码问题，特殊字符现在能正确显示。

**改进：**

1.  **迁移 CLI**：新增 `docmd migrate --upgrade`，实现配置自动现代化。
2.  **界面优化**：重新设计项目切换器样式，与 i18n 和外观按钮保持视觉一致。
