## 🐛 错误修复

### 插件自动安装:对 `import`-only `exports` 具有弹性

`@docmd/api` 中的自动安装器原先使用 Node 的 CommonJS `require.resolve`
来解析已安装的包,然后用 `await import(file://path)` 加载解析后的路径。
对于在 `exports` 字段中只声明 `import` 条件的包,`require.resolve` 会
抛出 `ERR_PACKAGE_PATH_NOT_EXPORTED`,即使该包已正确安装,构建仍会
打印:

```
⬢ Could not load @docmd/template-summer after auto-install
```

重试路径现在直接使用 `await import(name)`,它会原生识别 `exports` 字
段,并对任何条件均能正常工作。

重试路径内部还增加了注册表复核(纵深防御)。可自动安装的*名称*集合
保持不变 —— 仅扩展了它们可携带的 `exports` 条件集合。

### `@docmd/template-summer@0.8.9`:新增 `default` exports 条件

`@docmd/template-summer` 此前发布的 `exports["."]` 只声明了 `import`
条件。即使该包已正确安装,加载器中第一次 `require.resolve` 仍会因
此失败。该包现在声明:

```json
"exports": {
  ".": {
    "types": "./dist/index.d.ts",
    "import": "./dist/index.js",
    "default": "./dist/index.js"
  }
}
```

这与 `@docmd/plugin-git`、`@docmd/plugin-openapi` 和
`@docmd/plugin-threads` 的约定保持一致。在将 `@docmd/core` 升级到
`0.8.9` 后,请更新您的锁文件 (`npm install`、`pnpm install`、
`yarn install` 或 `bun install`)。

### `docmd-search@0.1.0`:稳定版,不再是 alpha,并带 `default` exports

`docmd-search` 不再带 `alpha` 标签。`0.1.0` 版本:

- 去掉了 `-alpha.1` 后缀。`latest` 标签对应的版本现在是 `0.1.0`。两个
  `0.1.0-alpha.*` 版本已弃用 —— `npm install docmd-search` 会按预期
  解析到 `0.1.0`。
- 在 `exports` 的 `.` 和 `./client` 子路径中都增加了
  `"default": "./dist/index.js"`。
- 移除了发布 workflow 中的 `--tag alpha` 标志。所有未来的发布都会落
  在默认的 `latest` 标签上。

如果您在文档中使用了语义搜索(`plugins.search.semantic = true`),
请更新您的 `node_modules`:

```bash
rm -rf node_modules/docmd-search
npm install         # 或 pnpm/yarn/bun install
```

### 插件加载器:加固的 `resolveDocmdSearch` 回退链

搜索插件解析可选的 `docmd-search` 同行依赖时,现在在读取清单时使用
显式的 `import` → `default` → `require` → `main` 回退链,并在
`./package.json` 子路径导出被移除或 pnpm 隔离的 `node_modules` 布局
造成解析器混淆时,以手动遍历 `node_modules` 作为后备。该函数现在能
适应常见的安装布局变化,而返回值契约保持不变。

## 🔒 安全性

没有新增攻击面。自动安装的信任边界仍然是官方插件注册表允许列表加
上 `@docmd/plugin-*` / `@docmd/template-*` 前缀检查。新的
`import(name)` 重试路径并未扩展可安装包的集合 —— 它仅扩展了注册表
批准的包可以携带的 `exports` 条件集合。

本地路径插件的安全路径(`safePath`,CWE-22/CWE-94)保持不变。重试路
径内部新增的注册表复核是额外的保障,不是放宽限制。

## 📚 文档更新

- **插件 → 使用插件** — "自动安装"小节现在说明弹性解析器和注册表
  复核。
- **插件 → 搜索** — "自动安装"小节提到加固后的 `resolveDocmdSearch`
  回退链。
- **开发 → 构建插件** 和 **开发 → 构建模板** — 新增 "ESM exports —
  `default` 条件"子小节,告知作者必须在 `exports` 中加入
  `"default": "./dist/index.js"` 才能让自动安装路径行为整洁。
